### 2. 프로젝트 개요 공유 (20분) — PM 발표

> 📎 참고 자료: `A2팀_서비스기획서_초안_20260626.md`

**2-1. 서비스 한 줄 정의 (2분)**
> "방송 제작자가 텍스트 기획 하나로 이미지·영상을 생성하고, 레퍼런스를 역분석해 프롬프트를 추출하는 AI 생성 엔진"

**2-2. A/B팀 역할 구분 (3분)**

| 방향 | 기능 | 담당 |
|------|------|------|
| 텍스트 → 이미지 | 이미지 생성 | **A2팀 (우리)** |
| 텍스트 → 영상 | 텍스트·키이미지 → 영상 클립 | **A2팀 (우리)** |
| 이미지 → 프롬프트 | 역프롬프트 추출 | **A2팀 (우리)** |
| 이미지 → 스토리보드 | 9컷 그리드 생성 + 샷별 텍스트 추출 | **B팀** |

**2-3. 핵심 기능 4가지 설명 (10분)**
- F001 이미지 생성
- F003 영상 생성 (비동기)
- F004 역프롬프트
- F005 미디어 라이브러리

> **범위 참고:** 스토리보드 그리드(9컷) 생성은 B팀 담당. A2팀은 이미지·영상 생성 및 역프롬프트에 집중.

**2-4. 전체 워크플로우 흐름 (5분)**
```
[A2팀] 텍스트 → 이미지·영상 생성 → 라이브러리 저장
                역프롬프트 추출

[사용자] A2팀에서 이미지 로컬 저장 → B팀 서비스에 직접 업로드
[B팀]   이미지 → 9컷 스토리보드 그리드 + 샷별 프롬프트 추출

[C팀]   음악·자막 후처리
```

### 3. 기술 스택 확인 및 의견 수렴 (30분)

**3-1. BE에게 확인할 사항 (15분)**

| 항목 | 가안 | 확인 포인트 |
|------|------|-----------|
| 이미지 API | GPT Image 2 단일 모델 확정 | 구현 방식 확인 (quality: medium 테스트 / high 데모) |
| 영상 API | fal.ai Seedance 2.0 / Kling 3.0 | 비동기 polling 방식 경험 있는지 |
| DB·스토리지 | Supabase | 사용 경험 여부, 대안 의견 |
| 역프롬프트 | Claude Sonnet API | 멀티모달 이미지 분석 → 영문 프롬프트 추출 (F004) |

- fal.ai queue → status polling → result 흐름 아는지 확인
- Supabase Storage(파일)·DB(메타데이터) 역할 분리 방식 공유

**3-2. FE에게 확인할 사항 (8분)**

| 항목 | 확인 포인트 |
|------|-----------|
| Next.js 14 App Router | 경험 여부, 어려운 부분 |
| 비동기 상태 관리 | 영상 생성 중 polling UI 구현 방식 논의 |
| 공통 컴포넌트 계획 | 탭 구조, 결과 그리드, 업로드 컴포넌트 |

**3-3. UX/UI에게 확인할 사항 (7분)**

| 항목 | 확인 포인트 |
|------|-----------|
| 디자인 시스템 | PD 레퍼런스 공유 받았는지, 없으면 임시로 시작 |
| 레퍼런스 서비스 | Krea·Higgsfield UI 함께 확인 |
| 1주차 산출물 | 무드보드·와이어프레임 방향 합의 |
