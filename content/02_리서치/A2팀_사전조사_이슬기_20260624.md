## 2. A·B팀 역할 구분

킥오프 회의록과 별도로, PM 레벨에서 기능 경계를 명확히 정리한 내용.

| 방향 | 기능 | 담당 |
|------|------|------|
| 텍스트 → 이미지 | 키이미지 생성 | **A팀** |
| 텍스트 → 이미지 | 9컷 스토리보드 그리드 이미지 생성 | **A팀** |
| 텍스트 → 영상 | 스토리보드 텍스트 기반 영상 클립 생성 | **A팀** |
| 이미지/영상 → 텍스트 | 레퍼런스 기반 역프롬프트 추출 | **A팀** |
| 이미지 → 텍스트 | 9컷 그리드 이미지 → 샷별 스토리보드 텍스트 추출 | **B팀** |

> **핵심 구분:** A팀은 텍스트→미디어(생성) 방향 + 역프롬프트(미디어→텍스트)를 담당. B팀은 A가 만든 이미지를 받아 텍스트 스토리보드로 변환하는 역방향 담당.

## 4. 레퍼런스 서비스 분석

PD가 명세서에서 언급한 Krea·Magnific·Higgsfield 세 서비스를 A2팀 관점에서 분석.

### 4.1 Krea.ai
- **핵심:** 실시간 이미지 생성 캔버스 + 64개 모델 통합 플랫폼
- **UX 특징:** 스케치하면 50ms 이내 실시간 렌더링 (Realtime Canvas)
- **우리가 참고할 것:** 모델 선택 드롭다운 UI, 생성 결과 그리드 레이아웃
- **우리가 안 해도 되는 것:** 실시간 렌더링, 인페인팅 편집

### 4.2 Magnific.ai
- **핵심:** AI 이미지 업스케일러 (저해상도 → 최대 10K)
- **UX 특징:** 슬라이더(Creativity/HDR/Resemblance) + 용도별 프리셋 조합
- **우리가 참고할 것:** 옵션 UI 설계 방식(슬라이더+프리셋), Before/After 비교 뷰 → 역프롬프트 결과 비교에 응용 가능
- **우리가 안 해도 되는 것:** 업스케일 기능 자체

### 4.3 Higgsfield.ai
- **핵심:** 영화적 카메라 프리셋 + 캐릭터 일관성(Soul ID)이 강점인 영상 생성 플랫폼
- **UX 특징:** 70개+ 카메라 프리셋(돌리샷·크레인·FPV 드론 등)을 버튼으로 선택
- **우리가 참고할 것:** 영상 생성 옵션 UI(길이·카메라 무브먼트), 비동기 처리 중 진행 상태 표시
- **우리가 안 해도 되는 것:** Soul ID 수준의 캐릭터 일관성 (MVP 범위 외)

### 4.4 A2팀 포지셔닝
> "Krea의 이미지 생성 + Higgsfield의 영상 생성 플로우를 **방송 제작자(PD·AD) 관점**으로 단순화한 올인원 생성·보관 도구"
>
> 세 서비스 모두 일반 크리에이터 대상이지만, **우리는 방송 제작자가 타겟** → 프롬프트 가이드, 역프롬프트, 라이브러리 정리가 차별점

## 5. 기술 가능성 분석

### 5.1 9컷 스토리보드 그리드 (캐릭터 일관성 포함)

**가능 여부: ✅ 가능 (~85% 일관성)**

- GPT Image 2의 **9-panel grid 방식**이 2026년 현재 가장 검증된 방법
- 하나의 캔버스 위 9개 영역으로 생성하므로 자연스럽게 캐릭터·의상·세계관이 일관되게 유지됨
- 캐릭터 시트(정면·측면·후면)를 미리 만들어 레퍼런스로 주입하면 정확도 향상 가능
- 한계: 빠른 동작 장면에서 얼굴 드리프트 발생 가능, 100% 일관성 보장 불가

**구현 흐름:**
```
텍스트 기획 입력
    → Claude API: 9컷 씬 분리 + 앵글 지시어 + 영문 프롬프트 생성
    → GPT Image 2: 9-panel grid 이미지 1장 생성
```

### 5.2 스토리보드 텍스트 → 완성 영상

**가능 여부: ✅ 가능 (클립 단위 생성 후 이어붙임)**

- 단일 API 호출로 긴 영상 생성은 불가능 (모델별 클립 최대 길이 존재)
- 씬별로 클립을 각각 생성한 후 ffmpeg로 이어붙이는 방식으로 구현

| 모델 | 클립 최대 길이 | 비용/초 |
|------|-------------|--------|
| Seedance 2.0 (fal.ai) | 15초 | $0.30 |
| Kling 3.0 Pro (fal.ai) | 10초 | $0.11 |

**구현 흐름:**
```
스토리보드 텍스트 (B팀 산출물)
    → GPT Image 2: 씬별 키이미지(첫 프레임) 생성
    → Seedance 2.0: 키이미지 + 프롬프트 → 영상 클립 (image-to-video)
    → ffmpeg: 클립 이어붙이기 → 완성 영상
```

### 5.3 영상 길이 제한

- **단일 API 호출로 무제한 영상: 불가**
- **클립 체이닝으로 사실상 무제한: 가능** (단, 비용·시간 선형 증가)
- 30분 영상 기준 예상 비용: Kling 3.0 ~$198 / Seedance 2.0 ~$540
- **현업 MVP 범위에서는 씬 단위 클립 생성 + 수동 or 자동 이어붙임으로 충분**

## 6. 이미지 모델 비교: GPT Image 2 vs Gemini

### 6.1 항목별 비교

| 항목 | GPT Image 2 | Gemini 2.5 Flash Image |
|------|------------|----------------------|
| 이미지 품질 | ⭐⭐⭐⭐⭐ 디테일·사실감 우수 | ⭐⭐⭐⭐ 추상·페인팅 스타일에 강점 |
| 프롬프트 정확도 | ⭐⭐⭐⭐⭐ 복잡한 다중 구성에 강함 | ⭐⭐⭐⭐ 단순 요청은 충분 |
| 한국어 지원 | ✅ 한글 렌더링 공식 지원 | ✅ 지원하나 정확도 낮음 |
| 캐릭터 일관성 | ✅ 9-panel grid 검증 완료 | ⚠️ 멀티 패널에서 다소 불안정 |
| 속도 | 🐢 생성 1분 내외 | 🐇 3~5배 빠름 (약 12초) |
| 비용 (1024×1024) | $0.07~0.167/장 | $0.039/장 |
| 9컷 그리드 적합성 | ✅ 최적 | ⚠️ 단순 구성만 가능 |

### 6.2 기능별 추천 모델

| 기능 | 개발·테스트 | 데모·최종 출력 |
|------|-----------|-------------|
| F001 키이미지 생성 | Gemini (빠르고 저렴) | GPT Image 2 (품질 우선) |
| F002 9컷 그리드 | GPT Image 2 (캐릭터 일관성 필수) | GPT Image 2 |
| F004 역프롬프트 | Gemini 2.5 Flash (멀티모달 텍스트 분석) | Gemini 2.5 Flash |

### 6.3 킥오프 액션 아이템 #5 검토 결과

> PD 요청: "GPT 이미지를 메인으로 갈 수 있는지 검토"

**결론: GPT Image 2 메인 채택 권장 + Gemini 보조 이원화**

```
GPT Image 2  → F001 (최종 키이미지), F002 (9컷 그리드)   — 품질 우선
Gemini Flash → F001 (개발 테스트 전용), F004 (역프롬프트)  — 속도·비용 절감
```
