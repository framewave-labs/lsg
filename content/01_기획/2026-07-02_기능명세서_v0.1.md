## 1. 로그인·회원가입

### 1-1. 회원가입

| 항목 | 내용 |
|------|------|
| 기능 ID | AUTH-001 |
| 화면 | 회원가입 화면 |

**사용자 입력**
- 이메일
- 비밀번호
- 비밀번호 확인

**시스템 동작**
1. 이메일 형식 유효성 검사
2. 비밀번호 최소 조건 검사 (8자 이상)
3. 비밀번호·비밀번호 확인 일치 여부 확인
4. 중복 이메일 확인
5. 계정 생성 완료 → 로그인 화면으로 이동

**에러 처리**
- 이메일 형식 오류 → "올바른 이메일 형식을 입력해주세요"
- 비밀번호 조건 미충족 → "비밀번호는 8자 이상이어야 합니다"
- 비밀번호 불일치 → "비밀번호가 일치하지 않습니다"
- 중복 이메일 → "이미 사용 중인 이메일입니다"

## 2. 이미지 생성 (M001~M006)

### 2-1. 프롬프트 입력 및 옵션 선택 (M001~M002)

| 항목 | 내용 |
|------|------|
| 기능 ID | M001, M002 |
| 화면 | 이미지 생성 화면 |
| API | `POST /generate-image` |

**사용자 입력**
- 프롬프트 텍스트 (필수)
- 이미지 비율: 1:1 / 16:9 / 9:16 / 4:3 (기본값: 1:1)
- 품질: medium (기본) / high (데모 시)

**시스템 동작**
1. 프롬프트 입력값 검증 (빈값 방지)
2. GPT Image 2 API 호출 (quality, size 파라미터 포함)
3. 생성 중 로딩 상태 표시
4. 결과 이미지 반환 → 결과 화면으로 전환

**에러 처리**
- 프롬프트 빈값 → "프롬프트를 입력해주세요" (생성 버튼 비활성화)
- API 호출 실패 → "이미지 생성에 실패했습니다. 다시 시도해주세요" + 재시도 버튼
- 타임아웃 (60초 초과) → 에러 메시지 + 재시도

## 3. 영상 생성 (A002~A005)

### 3-1. 프롬프트 입력 및 옵션 선택 (A002)

| 항목 | 내용 |
|------|------|
| 기능 ID | A002 |
| 화면 | 영상 생성 화면 |
| API | `POST /generate-video` |

**사용자 입력**
- 프롬프트 텍스트 (필수)
- 모델: Kling 3.0 (테스트) / Seedance 2.0 (품질 확인·데모)
- 영상 길이: 5초 / 10초 / 15초
- 해상도: 480p / 720p

**시스템 동작**
1. 프롬프트 입력값 검증
2. fal.ai API 호출 → request_id 수신
3. 생성 중 상태 화면으로 전환

**에러 처리**
- 프롬프트 빈값 → 생성 버튼 비활성화
- API 호출 실패 → "영상 생성 요청에 실패했습니다. 다시 시도해주세요"

---

### 3-2. 영상 생성 상태 조회 (A003)

| 항목 | 내용 |
|------|------|
| 기능 ID | A003 |
| 화면 | 영상 생성 화면 — 생성 중 상태 |
| API | `GET /video-status?request_id={id}` |

**시스템 동작**
1. 5초 간격 polling으로 생성 상태 조회
2. 상태값에 따라 UI 업데이트:
   - `IN_QUEUE` → "대기 중..."
   - `IN_PROGRESS` → "생성 중..." + 진행 표시
   - `COMPLETED` → 결과 화면으로 전환
   - `ERROR` → 에러 메시지 + 재시도 버튼

**에러 처리**
- 타임아웃 (10분 초과) → "생성 시간이 초과됐습니다. 다시 시도해주세요"
- ERROR 상태 수신 → "영상 생성에 실패했습니다. 다시 시도해주세요"

## 5. 역프롬프트 (S001~S004)
> S등급 도전 기능 / 진입점 위치는 IA 확정 후 반영 예정

### 5-1. 이미지 업로드 및 분석 (S001~S002)

| 항목 | 내용 |
|------|------|
| 기능 ID | S001, S002 |
| 화면 | 역프롬프트 화면 |
| API | `POST /reverse-prompt` |

**사용자 입력**
- 레퍼런스 이미지 업로드 (jpg·png·webp)

**시스템 동작**
1. 이미지 업로드 후 GPT-4o Vision API 호출
2. JSON 형태로 분석 결과 반환:
   ```
   analysis.style
   analysis.lighting
   analysis.composition
   analysis.lens
   analysis.color_palette
   analysis.mood
   prompt_en (80단어 이하)
   ```
3. 분석 항목을 태그 형태로 시각화
4. `prompt_en` 복사 버튼 제공

**에러 처리**
- 지원하지 않는 파일 형식 → "jpg, png, webp 파일만 업로드 가능합니다"
- 파일 용량 초과 → "10MB 이하 파일만 업로드 가능합니다"
- API 분석 실패 → "이미지 분석에 실패했습니다. 다시 시도해주세요"

---
