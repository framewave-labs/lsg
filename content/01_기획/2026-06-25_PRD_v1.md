## 1. 제품 한 줄 정의 & 범위

**KX GenStudio** — 방송 제작자가 텍스트 기획으로 이미지·영상을 생성하고 라이브러리에 보관하는 AI 생성 엔진

### MVP 기능 범위

| 기능 ID | 기능명 | 우선순위 | 담당 주차 |
|--------|-------|---------|---------|
| F001 | 이미지 생성 | 필수 | 2주차 |
| F003-B | 영상 생성 기본 (단일 Shot) | 필수 | 3주차 |
| F005 | 미디어 라이브러리 | 필수 | 2주차 |
| F003-A | Shot 세션 관리 + 개별 재생성 | 도전 | 3주차 후반 |
| F004 | 역프롬프트 4단계 (추출→수정→재생성→저장) | 도전 (S등급) | 4주차 |

### 등급별 달성 기준

| 등급 | 핵심 달성 조건 |
|------|-------------|
| **B** | F001 + F003-B(단일 Shot 생성·저장) + F005 + 배포 |
| **A** | F001 + F003-A(Shot 세션·재생성) + F005 + 배포 |
| **S** | A등급 + F004 역프롬프트 4단계(추출→수정→재생성→저장) |

> **F003 상한선:** F003은 A등급(Shot 세션 관리)까지. S등급 차별점은 F004 역프롬프트 완성도.

> **범위 확정 (2026-06-26):** F002 스토리보드 그리드 생성은 B팀 범위. A2팀에서 구현하지 않음.
> **구조 확정 (2026-06-26):** F003 입력 = B팀 Shot 텍스트 프롬프트(Shot1~9) + B팀 9분할 그리드 이미지. 캐릭터 일관성은 그리드 이미지가 레퍼런스 앵커 역할.

## 2. 유저 스토리 & 인수 조건

### F001 — 이미지 생성

#### US-01: 기본 이미지 생성
```
As 방송 AD,
I want to type a prompt and generate an image,
So that I can quickly visualize a scene without knowing AI tools.
```

**인수 조건 (Acceptance Criteria)**
- [ ] 프롬프트 입력창에 텍스트를 입력하고 [생성하기] 버튼을 클릭하면 이미지가 생성된다
- [ ] 생성 중에는 로딩 상태가 표시되며 버튼은 비활성화된다
- [ ] 생성 완료 후 결과 이미지가 화면에 표시된다
- [ ] 생성된 이미지는 자동으로 라이브러리에 저장된다
- [ ] 빈 프롬프트로 제출 시 생성 요청이 발생하지 않고 안내 문구가 표시된다

#### US-02: 모델·비율 선택
```
As 방송 PD,
I want to choose between image models and aspect ratios,
So that I can optimize for quality or speed based on the situation.
```

**인수 조건**
- [ ] 이미지 API는 GPT Image 2 단일 모델을 사용한다
- [ ] 비율 선택: 1:1 / 16:9 / 9:16 중 선택 가능
- [ ] 선택한 비율이 생성 결과에 반영된다
- [ ] 결과 이미지 카드에 생성일이 표시된다

---

### F003 — 영상 생성

#### US-03: 영상 비동기 생성
```
As 방송 PD,
I want to generate a video clip from a text prompt,
So that I can preview a scene in motion before actual shooting.
```

**인수 조건**
- [ ] 프롬프트 입력 + 모델·길이 선택 후 [영상 생성] 버튼 클릭 시 생성 요청이 제출된다
- [ ] 생성 중 상태("대기 중" → "생성 중" → "완료")가 화면에 표시된다
- [ ] 완료 후 인라인 비디오 플레이어에서 영상을 바로 재생할 수 있다
- [ ] 생성된 영상은 자동으로 라이브러리에 저장된다
- [ ] 10분 경과 후 완료되지 않으면 타임아웃 에러 메시지와 재시도 버튼이 표시된다

#### US-04: 영상 모델·길이 선택
```
As 방송 PD,
I want to choose video model and duration,
So that I can balance between cost and quality.
```

**인수 조건**
- [ ] 모델 선택: Kling 3.0 (기본, 테스트용) / Seedance 2.0 (고품질) 중 선택 가능
- [ ] 영상 길이: 5초 / 10초 / 15초 중 선택 가능
- [ ] 기본값: Kling 3.0, 5초 (비용 통제)

---

### F003-A — Shot 세션 관리 (A등급 도전 과제)

#### US-08: Shot 세션 생성 및 개별 재생성
```
As 방송 PD,
I want to manage Shot1~9 as a session and regenerate specific shots,
So that I can refine individual clips without redoing the entire storyboard.
```

**인수 조건**
- [ ] B팀 9분할 그리드 이미지 + Shot1~9 텍스트 프롬프트를 업로드하면 세션이 생성된다
- [ ] 세션 화면에 Shot1~9 카드가 순서대로 나열되고 각각 생성 상태가 표시된다
- [ ] [전체 생성] 버튼으로 Shot1~9를 순차적으로 일괄 생성 요청할 수 있다
- [ ] 각 Shot 카드에 [재생성] 버튼이 있어 해당 Shot만 단독으로 재생성할 수 있다
- [ ] 재생성 중에도 다른 Shot의 영상은 영향받지 않는다
- [ ] 완료된 Shot 카드에서 영상을 인라인으로 미리볼 수 있다

### F005 — 미디어 라이브러리

#### US-05: 생성물 보관 및 조회
```
As 방송 AD,
I want to see all generated images and videos in one place,
So that I can manage and reuse my creative assets.
```

**인수 조건**
- [ ] 라이브러리 탭에 이미지·영상이 생성 최신순으로 그리드 형태로 표시된다
- [ ] 이미지/영상 타입 필터로 원하는 타입만 볼 수 있다
- [ ] 아이템을 클릭하면 상세 모달이 열린다
- [ ] 모달에서 다운로드 버튼으로 파일을 저장할 수 있다
- [ ] 아이템이 없을 때 빈 상태 UI와 생성 시작 안내가 표시된다
- [ ] 20개 초과 시 추가 로드(페이지네이션 또는 무한스크롤)가 동작한다

### F004 — 역프롬프트 (S등급 도전 과제)

#### US-07: 레퍼런스 이미지 → 프롬프트 추출 → 수정 → 재생성 → 저장

```
As 방송 PD,
I want to upload a reference image, extract a prompt, refine it, and regenerate a new image on the same page,
So that I can recreate a similar style without switching tools.
```

**인수 조건 — 1단계: 추출**
- [ ] 이미지 파일(JPG/PNG, 최대 10MB)을 업로드할 수 있다
- [ ] [프롬프트 추출] 클릭 시 Claude Sonnet이 분석해 스타일·조명·구도·렌즈·색감·분위기 항목이 담긴 영문 프롬프트를 반환한다
- [ ] 추출된 프롬프트는 복사 버튼과 함께 표시된다

**인수 조건 — 2단계: 수정**
- [ ] 추출된 프롬프트를 텍스트 에디터에서 직접 수정할 수 있다
- [ ] 수정 중에도 원본 추출 프롬프트를 확인할 수 있다 (원본 보기 토글)

**인수 조건 — 3단계: 재생성**
- [ ] [이 프롬프트로 이미지 생성] 버튼 클릭 시 동일 페이지에서 GPT Image 2로 이미지를 생성한다
- [ ] 생성 중 로딩 상태가 표시된다
- [ ] 생성 완료 후 결과 이미지가 동일 페이지에 표시된다

**인수 조건 — 4단계: 저장**
- [ ] 생성된 이미지가 자동으로 라이브러리에 저장된다
- [ ] 저장 완료 후 라이브러리 이동 링크가 표시된다

## 3. 데이터 모델 (Supabase 스키마)

### 3.1 테이블: `media_items`

이미지와 영상을 단일 테이블로 관리합니다.

```sql
CREATE TABLE media_items (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  type        TEXT NOT NULL CHECK (type IN ('image', 'video')),
  status      TEXT NOT NULL DEFAULT 'completed'
              CHECK (status IN ('pending', 'processing', 'completed', 'failed')),
  
  -- 생성 정보
  prompt      TEXT,                        -- 사용자 입력 프롬프트
  model       TEXT,                        -- 사용한 API 모델명
  aspect_ratio TEXT,                       -- 이미지 비율 (1:1, 16:9, 9:16)
  duration    INTEGER,                     -- 영상 길이 (초, 영상만)
  
  -- 파일 정보
  storage_path TEXT,                       -- Supabase Storage 경로
  public_url  TEXT,                        -- 공개 접근 URL
  thumbnail_url TEXT,                      -- 썸네일 URL (영상만)
  file_size   INTEGER,                     -- 파일 크기 (bytes)
  
  -- 비동기 처리 (영상만)
  fal_request_id TEXT,                     -- fal.ai queue request_id
  
  -- 메타데이터
  created_at  TIMESTAMPTZ DEFAULT NOW(),
  updated_at  TIMESTAMPTZ DEFAULT NOW()
);

-- 최신순 조회용 인덱스
CREATE INDEX idx_media_items_created_at ON media_items(created_at DESC);
CREATE INDEX idx_media_items_type ON media_items(type);
```

### 3.2 테이블: `storyboard_sessions` (A등급~)

Shot 세션 관리용. Shot1~9를 하나의 작업 단위로 묶습니다.

```sql
CREATE TABLE storyboard_sessions (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  grid_image_url  TEXT NOT NULL,   -- B팀 9분할 그리드 이미지 URL
  status          TEXT NOT NULL DEFAULT 'in_progress'
                  CHECK (status IN ('in_progress', 'completed')),
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  updated_at      TIMESTAMPTZ DEFAULT NOW()
);
```

### 3.3 테이블: `storyboard_shots` (A등급~)

세션 내 개별 Shot 상태 및 결과를 관리합니다.

```sql
CREATE TABLE storyboard_shots (
  id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id     UUID NOT NULL REFERENCES storyboard_sessions(id) ON DELETE CASCADE,
  shot_number    INTEGER NOT NULL CHECK (shot_number BETWEEN 1 AND 9),
  prompt         TEXT NOT NULL,    -- B팀 제공 Shot 텍스트 프롬프트
  status         TEXT NOT NULL DEFAULT 'pending'
                 CHECK (status IN ('pending', 'processing', 'completed', 'failed')),
  video_url      TEXT,
  fal_request_id TEXT,
  created_at     TIMESTAMPTZ DEFAULT NOW(),
  updated_at     TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE (session_id, shot_number)
);

CREATE INDEX idx_storyboard_shots_session ON storyboard_shots(session_id, shot_number);
```

```
버킷: kx-genstudio
  ├── images/
```

### 3.3 데이터 흐름

**이미지 생성 시:**
```
1. POST /api/generate-image 호출
2. API에서 이미지 생성 성공
3. Supabase Storage에 이미지 업로드 → storage_path, public_url 획득
4. media_items 테이블에 INSERT (status: 'completed')
5. FE에 { id, public_url } 반환
```

**영상 생성 시:**
```
1. POST /api/generate-video 호출
2. fal.ai queue.submit() → request_id 획득
3. media_items에 INSERT (status: 'processing', fal_request_id: request_id)
4. FE에 { id, requestId, status: 'processing' } 반환
5. FE가 5초 간격으로 GET /api/video-status?id={id} 폴링
6. 완료 시: fal.ai에서 영상 URL 획득 → Storage에 저장 → media_items UPDATE (status: 'completed')
7. FE에 { status: 'completed', public_url } 반환
```

## 4. API 상세 명세

> fal.ai 비동기 흐름 상세 및 Claude 시스템 프롬프트는 `A2팀_BE공유_기술기획_20260625.md` 참조

### 4.1 `POST /api/generate-image`

**Request**
```typescript
{
  prompt: string        // 필수, 1자 이상
  model: 'gpt-image-2'                    // 단일 모델 고정
  aspectRatio: '1:1' | '16:9' | '9:16'   // 기본값: '1:1'
}
```

**Response — 성공 (200)**
```typescript
{
  success: true,
  data: {
    id: string           // media_items.id
    public_url: string   // 이미지 URL
    model: string
    aspect_ratio: string
    created_at: string
  }
}
```

**Response — 실패 (400 / 500)**
```typescript
{
  success: false,
  error: {
    code: 'INVALID_INPUT' | 'GENERATION_FAILED' | 'API_LIMIT_EXCEEDED',
    message: string      // 사용자에게 표시할 메시지 (한국어)
  }
}
```

**GPT Image 2 API 호출 참고**
```typescript
import OpenAI from 'openai';
const openai = new OpenAI();

const response = await openai.images.generate({
  model: 'gpt-image-1',
  prompt: prompt,
  size: aspectRatio === '16:9' ? '1792x1024'
       : aspectRatio === '9:16' ? '1024x1792'
       : '1024x1024',
  quality: 'medium',    // 테스트: medium / 데모: high
  output_format: 'png',
});
// response.data[0].b64_json → base64 이미지
```


### 5.4 상태 관리 방침

- **모달 열림/닫힘 상태**는 `page.tsx` 레벨 `useState`로 관리 — 동시에 하나의 모달만 열림
- **영상 폴링 상태**는 `VideoGenerateModal` 내부 `useState` + `useEffect`로 관리
- **라이브러리**는 LibraryModal 열릴 때 API 호출로 최신 데이터 패치

```typescript
// page.tsx 모달 상태 관리 예시
type ModalType = 'image' | 'video' | 'storyboard' | 'reversePrompt' | 'library' | null;
const [activeModal, setActiveModal] = useState<ModalType>(null);

const openModal = (type: ModalType) => setActiveModal(type);
const closeModal = () => setActiveModal(null);
```

```typescript
// 영상 폴링 예시 (FE 참고)
const [videoStatus, setVideoStatus] = useState<'idle' | 'processing' | 'completed' | 'failed'>('idle');
const [mediaId, setMediaId] = useState<string | null>(null);

useEffect(() => {
  if (!mediaId || videoStatus !== 'processing') return;

  const interval = setInterval(async () => {
    const res = await fetch(`/api/video-status?id=${mediaId}`);
    const { data } = await res.json();
    
    if (data.status === 'completed' || data.status === 'failed') {
      setVideoStatus(data.status);
      clearInterval(interval);
    }
  }, 5000);

  // 10분 타임아웃
  const timeout = setTimeout(() => {
    setVideoStatus('failed');
    clearInterval(interval);
  }, 600000);

  return () => { clearInterval(interval); clearTimeout(timeout); };
}, [mediaId, videoStatus]);
```
