## 페이지 구조 — 통합 1페이지로 결정

**레퍼런스 조사 결과**: Krea·Pika·Magnific은 계정/설정을 한 페이지로 통합, Higgsfield·Krea Omni는 공개 프로필(팔로워·발행작품)과 계정 설정을 분리.
**결정**: 우리 서비스는 팔로우·좋아요·발행 같은 소셜 기능이 없고(홈 "탐색하기"는 추천·큐레이션 갤러리일 뿐, 유저 발행 콘텐츠 아님) 분리할 이유가 없음 → **통합 1페이지**로 진행.

와이어프레임은 로그인 방식(Google/일반)별 차이를 명확히 보여주기 위해 좌측 통합 페이지 + 우측 방식별 비교표, 두 형태를 함께 구성.

| 속성 | 내용 | 비고 |
|------|------|------|
| 로그인 방식 뱃지 | "일반" / "Google" | 가입 방식에 따라 하위 항목 분기 |
| 비밀번호 변경 | 일반 가입자만 노출 | Google 로그인 유저는 항목 자체 숨김 |

| 속성 | 내용 | 비고 |

### 회원 탈퇴 확인 모달 — 레퍼런스 반영

Freepik/Slidesgo 계열 삭제 모달 패턴을 참고해 구조 확정:

1. 제목: "계정을 삭제하려고 합니다"
2. 안내 문구: "계정이 영구적으로 삭제됩니다:"
3. 삭제 항목 체크리스트 (불릿)
   - 모든 생성물 접근 권한
   - 라이브러리 저장 콘텐츠
   - 저장된 설정
4. 경고 한 줄: "이 작업은 취소할 수 없습니다."
5. 체크박스 게이트: "계정 삭제는 영구적이고 되돌릴 수 없음을 이해했습니다." — **체크 전에는 "계정 삭제하기" 버튼 비활성화(연한 색)**, 체크해야 활성화(빨간색)
6. 버튼: 취소하기(보조) / 계정 삭제하기(위험, 체크 게이트 적용)

## 이번 초안에서 제외한 항목

- **크레딧/사용량**: 완전 제외 (사용량 제한 자체를 없애는 방향 논의 중 — 별도 후속안건 추적)
- **내 생성 이력 / 라이브러리 바로가기**: 사이드 메뉴와 중복되어 제외 (v0.4)
- **생성 완료 알림 토글**: 현재는 제외, 필요시 재검토
- **세션/기기 관리, 언어·테마 설정**: 내부 서비스 특성상 우선순위 낮음, 후속 로드맵
