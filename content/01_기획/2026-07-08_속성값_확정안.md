# A2팀 UI 속성값 확정안
> 작성일: 2026-07-08 | 작성자: 이슬기 (PM)
> 목적: 디자이너·BE 공유용 — 각 화면의 사이드바 및 입력창 속성값 확정
> **Seedance 2.0 해상도 항목 일부 BE 확인 후 최종 확정 예정**

---

## 1. 홈 입력창

### 흐름
```
프롬프트 입력 → 칩 선택 → 엔터 → 선택한 메뉴 화면으로 이동 (프롬프트 자동 채워짐)
```

### 칩 구성

| 칩 | 이동 대상 |
|----|---------|
| 초기 레퍼런스 | 이미지 생성 메뉴 A |
| 키이미지 | 이미지 생성 메뉴 B |
| 영상 | 영상 생성 화면 |

> **칩을 최소화한 이유:** 홈 입력창은 진입점 역할만 한다. 세부 옵션(비율·품질 등)은 이동한 메뉴 화면에서 설정하므로 홈에서는 목적지 선택만 최소화하여 노출한다.

> **기본값:** 칩 선택 없이 엔터 시 → 초기 레퍼런스(메뉴 A)로 이동

---

## 2. 이미지 생성 — 메뉴 A (초기 레퍼런스)

> 캐릭터 및 배경의 초기 레퍼런스 이미지를 생성하는 화면. 레퍼런스 이미지를 직접 첨부해 스타일·캐릭터를 참조할 수 있도록 했다.

### 사이드바 속성값

| 속성 | 옵션 | API 파라미터 | 비고 |
|------|------|------------|------|
| 모델 | GPT Image 2 | — | 추후 모델 추가 가능성 고려, 현재는 단일 모델 표시 |
| 목적 | 캐릭터 / 배경 | 프롬프트 분기 | 선택값에 따라 생성 프롬프트 기본값 달라짐 |
| 레퍼런스 | Style + Character + Add (최대 8장) | 이미지 인풋 | PNG / WEBP / JPG, 50MB 이하 |
| 비율 | Auto / 1:1 / 16:9 / 9:16 | `size` | Auto: 모델이 컨텍스트에 맞게 자동 선택 |
| 품질 | 표준 / 고화질 | `quality: medium / high` | |
| 수량 | 1~4장 (+/- 스테퍼) | `n` | |
| AI 프롬프트 교정 | ON / OFF 토글 | Claude Sonnet 변환 레이어 | 한국어 → 최적화 영어 프롬프트 자동 변환 |

### 캐릭터 다각도 (정면·측면·후면) 출력 방식

캐릭터 목적 선택 시, 정면·측면·후면을 1장의 이미지에 담아 출력하는 방식을 적용한다.

- **처리 방법**: 프롬프트 엔지니어링으로 해결 — BE 별도 처리 불필요
- **적용 프롬프트 예시**:
  ```
  character design sheet showing front view, side view, and back view 
  arranged side by side, white background, photorealistic, high detail
  ```
- **수량 설정과의 관계**: 수량 1장 = 다각도 시트 1장 출력. 수량 2장 이상 선택 시 다각도 시트 여러 장 생성
- **주의**: GPT Image 2의 프롬프트 준수율에 의존 — 프롬프트 가이드 v2 업데이트 필요 (후속 작업)

## 3. 이미지 생성 — 메뉴 B (키이미지)

> 완성도 높은 캐릭터 시트 생성을 위한 화면. 메뉴 진입 자체가 키이미지 캐릭터시트 생성 확정이므로 목적 선택은 표시만 한다.

### 사이드바 속성값

| 속성 | 옵션 | API 파라미터 | 비고 |
|------|------|------------|------|
| 모델 | GPT Image 2 | — | 추후 모델 추가 가능성 고려, 현재는 단일 모델 표시 |
| 목적 | 키이미지 캐릭터시트 (표시만) | — | 선택 불필요, 메뉴 진입으로 확정 |
| 비율 | Auto / 1:1 / 16:9 / 9:16 | `size` | |
| 품질 | 표준 / 고화질 | `quality: medium / high` | PD님 파이프라인 고려, 사용자 선택 가능 |
| 수량 | 1장 고정 | `n: 1` | UI 미노출 |
| AI 프롬프트 교정 | ON / OFF 토글 | Claude Sonnet 변환 레이어 | |

### 속성 선택 항목

> 아래 항목들은 사이드바 내 속성 선택 영역에 노출. 선택값은 Claude Sonnet을 통해 최적화 영어 프롬프트로 자동 변환.

## 4. 영상 생성

### 사이드바 속성값

| 속성 | 옵션 | 비고 |
|------|------|------|
| 모델 | Seedance 2.0 (기본) / Kling 3.0 | 카드 형태로 선택 |
| 길이 | Seedance: 4s~15s / Kling: 3s~15s (1초 단위) | 모델 선택에 따라 FE 조건부 렌더링 — BE 추가 작업 없음 |
| 비율 | Seedance: Auto / 16:9 / 9:16 / 4:3 / 3:4 / 1:1 / 21:9 | Kling 선택 시 Auto / 16:9 / 9:16 / 1:1만 노출 |
| 해상도 | 720p (기본) / 1080p / 4K | 아래 해상도별 비용 참고 / Seedance 1080p·4K 지원 여부 BE 확인 필요 |
| Bitrate | High 고정 | UI 미노출 — BE에서 고정값 처리 |
| 스토리보드 | 9컷 3×3 그리드 업로드 | |
| AI 프롬프트 교정 | ON / OFF 토글 | Claude Sonnet 변환 레이어 |

### 해상도별 비용 (fal.ai 기준)

| 모델 | 해상도 | 비용 | 15초 영상 1개 기준 |
|------|--------|------|--------------------|
| Kling 3.0 Standard | 1080p | $0.084/초 | $1.26 (약 1,700원) |
| Kling 3.0 Pro | 1080p | $0.112/초 | $1.68 (약 2,300원) |
| Kling 3.0 | **4K** | $0.42/초 | $6.30 (약 8,700원) |
| Seedance 2.0 Mini | 480p | $0.043/초 | $0.65 (약 900원) |
| Seedance 2.0 | 720p | $0.303/초 | $4.55 (약 6,300원) |
| Seedance 2.0 | 1080p / 4K | **BE 확인 필요** | — |

### 데모 모드 vs 시연 모드 분리

UI에서 Bitrate를 노출하지 않고 BE 환경변수로 품질을 제어한다.

```
APP_ENV=demo      → 낮은 스펙 (빠른 테스트용)
APP_ENV=prod      → 고화질 (1080p, High Bitrate)
APP_ENV=showcase  → 최고 품질 (4K, 마지막 시연 날 활성화)
```

| 파라미터 | demo | prod | showcase |
|---------|------|------|----------|
| Seedance 해상도 | 480p | 720p | 1080p / 4K (BE 확인 후) |
| Kling 해상도 | 720p | 1080p | 4K |
| Bitrate | Standard | High | High |
