## UC-IMAGE-01. 메뉴 A — 초기 레퍼런스 생성

| 항목 | 내용 |
|------|------|
| 유즈케이스명 | 메뉴 A — 초기 레퍼런스 생성 |
| 유즈케이스 ID | UC-IMAGE-01 |
| 주 액터 | 방송 PD/AD |
| 보조 액터 | GPT Image 2 API, (AI 프롬프트 변환 On 시) Claude Sonnet 변환 레이어, AWS 스토리지 |
| 목표 | 간단한 한 줄 프롬프트로 캐릭터·배경의 초기 형태를 빠르게 확인한다 |
| Preconditions | 로그인 상태일 것 / 이미지 생성 화면(메뉴 A)에 진입해 있을 것 |
| Postconditions | 배경 1장 또는 캐릭터 다각도 이미지가 결과 영역에 표시됨 / [저장] 시 라이브러리에 반영됨 |
| 우선순위 | P0 (B등급 필수 — M001~M004) |
| 관련 문서 | IA v0.5 화면구조(메뉴 A), 기능명세서 v0.5 S4-1~4-3 |

**기본 흐름**
1. 사용자가 메뉴 A 탭으로 진입한다
2. 프롬프트 입력창에 캐릭터 또는 배경에 대한 간단한 설명을 입력한다 (한국어/영어 자유)
3. 비율(1:1 기본)과 품질(medium 기본)을 선택한다
4. [생성하기] 클릭 또는 Enter
5. 시스템이 배경 이미지 1장 또는 캐릭터 다각도(정면·측면·후면) 이미지를 생성한다 — **예상 소요 시간 10~15초**, 이 구간은 스피너/스켈레톤 + "이미지를 생성하고 있습니다…" 문구로 대기 상태를 명확히 알려준다 (막연한 로딩만 보여주면 사용자가 멈춘 것으로 오인할 수 있음)
6. 결과가 결과 영역에 표시된다
7. 사용자가 결과를 확인하고 [저장]으로 라이브러리에 보관하거나, 결과를 참고해 메뉴 B로 넘어간다

**대안/예외 흐름**
- 프롬프트가 빈값이면 [생성하기]가 비활성화 상태로 유지된다 (기본 흐름 4번 진입 불가)
- 생성 실패 시 에러 메시지와 [다시 시도]가 표시되고, 클릭 시 동일 입력값으로 재요청한다
- 생성 60초 초과 시 타임아웃 처리 후 [다시 시도] 노출 (예상 소요 시간의 4배 수준으로 여유를 둔 값)

## UC-IMAGE-02. 메뉴 B — 키이미지(캐릭터 시트) 생성

| 항목 | 내용 |
|------|------|
| 유즈케이스명 | 메뉴 B — 키이미지(캐릭터 시트) 생성 |
| 유즈케이스 ID | UC-IMAGE-02 |
| 주 액터 | 방송 PD/AD |
| 보조 액터 | GPT Image 2 API, (AI 프롬프트 변환 On 시) Claude Sonnet 변환 레이어, AWS 스토리지 |
| 목표 | 메뉴 A에서 확인한 캐릭터 외형을 바탕으로, 실제 제작에 쓸 완성도 높은 캐릭터 시트(턴어라운드+표정+소품+컬러 팔레트)를 만든다 |
| Preconditions | 로그인 상태일 것 / 이미지 생성 화면에서 메뉴 B 탭에 진입해 있을 것 (메뉴 A를 거치지 않고 바로 진입도 가능) |
| Postconditions | 캐릭터 키이미지가 결과 영역에 표시됨 / [저장] 시 라이브러리에 반영됨 |
| 우선순위 | P0 (B등급 필수 — M001~M004) |
| 관련 문서 | IA v0.5 화면구조(메뉴 B), 기능명세서 v0.5 S4-1~4-3, 메뉴AB 프롬프트가이드 v1 S3 |

**기본 흐름**
1. 사용자가 메뉴 B 탭으로 진입한다
2. 입력 방식을 고른다 — **자유 텍스트**(PART1 서술+PART2 구조화 필드) 또는 **속성 선택 사이드바**(파트1 캐릭터 기본~파트5 의상·악세서리)
3-A. (자유 텍스트) PD 표준형 구조에 맞춰 프롬프트를 직접 작성한다
3-B. (속성 선택) 사이드바에서 캐릭터 속성·헤어·얼굴·의상 값을 순서대로 선택한다. 컬러 항목은 팔레트+헥스코드로 직접 지정
4. [생성하기] 클릭 또는 Enter
5. 시스템이 캐릭터 키이미지(턴어라운드+표정 배리에이션+소품 클로즈업+컬러 팔레트)를 생성한다 — 메뉴 A보다 담을 요소가 많아 **예상 소요 시간 15~20초** 수준으로 안내 (제안, 실측 후 조정)
6. 결과가 결과 영역에 표시된다
7. 사용자가 [저장]으로 라이브러리에 보관한다 — 이후 영상 생성(UC-VIDEO-01)에서 이 캐릭터를 참고 레퍼런스로 활용

**대안/예외 흐름**
- 자유 텍스트 경로에서 프롬프트가 빈값이면 [생성하기] 비활성화
- 속성 선택 사이드바 경로에서 필수 항목 미입력 시 [생성하기] 비활성화
- 생성 실패·타임아웃 시 UC-IMAGE-01과 동일한 에러 처리

## UC-PROMPT-01. AI 프롬프트 변환 토글 사용

| 항목 | 내용 |
|------|------|
| 유즈케이스명 | AI 프롬프트 변환 토글 사용 |
| 유즈케이스 ID | UC-PROMPT-01 |
| 주 액터 | 방송 PD/AD (특히 영어 프롬프트 작성에 익숙하지 않은 사용자) |
| 보조 액터 | Claude Sonnet 변환 레이어, GPT Image 2 API |
| 목표 | 한국어로 입력해도 AI가 이미지 생성에 최적화된 영어 프롬프트로 바꿔주도록 해, 프롬프트 엔지니어링 지식 없이도 생성 품질을 확보한다 |
| Preconditions | 메뉴 A 또는 메뉴 B(자유 텍스트 입력 경로)에서 프롬프트 입력창에 텍스트가 입력되어 있을 것 |
| Postconditions | 저장된 결과의 prompt 메타데이터가 변환 On 시 변환된 영문 프롬프트 기준으로 기록됨 |
| 우선순위 | P1 (PM 제안 기능 — B등급 필수는 아니나 UX 핵심) |
| 관련 문서 | IA v0.5(전제조건 표), 기능명세서 v0.5 S4-1(공통 토글) |

**기본 흐름**
1. 사용자가 프롬프트 입력창에 한국어로 원하는 장면·캐릭터를 입력한다
2. "AI 프롬프트 변환" 토글을 On으로 켠다 (보조문구: "입력한 내용을 이미지 생성용 영어 프롬프트로 다듬어드려요")
3. [생성하기] 클릭 또는 Enter
4. 시스템이 백그라운드에서 Claude Sonnet으로 입력을 영어로 변환한다 — **변환된 영어 프롬프트는 화면에 노출하지 않는다**
5. 변환된 프롬프트로 바로 이미지 생성이 진행된다
6. 결과가 결과 영역에 표시된다

**대안/예외 흐름**
- 토글이 Off 상태면 입력한 텍스트를 그대로 생성 요청에 사용한다 (변환 단계 없음)
- 메뉴 B에서 속성 선택 사이드바 경로를 쓰는 경우 이 토글은 적용되지 않는다 (이미 구조화된 값이라 변환 대상 아님)
- 변환 자체가 실패해도 원본 입력 텍스트로 생성을 시도한다 (제안 — BE 협의 필요)

## UC-VIDEO-01. 영상 생성

| 항목 | 내용 |
|------|------|
| 유즈케이스명 | 영상 생성 |
| 유즈케이스 ID | UC-VIDEO-01 |
| 주 액터 | 방송 PD |
| 보조 액터 | fal.ai (Kling 3.0 / Seedance 2.0), AWS 스토리지, B팀(3x3 그리드 이미지 제공) |
| 목표 | B팀에서 받은 3x3 그리드 이미지와 영상 프롬프트로 짧은 영상 클립을 생성해 실제 촬영 전 장면을 미리 확인한다 |
| Preconditions | 로그인 상태일 것 / 영상 생성 화면에 진입해 있을 것 / B팀으로부터 받은 3x3 그리드 이미지를 보유하고 있을 것 |
| Postconditions | 생성된 영상이 결과 영역에 표시됨 / [저장] 시 라이브러리에 반영됨 |
| 우선순위 | P0 (B등급 필수 — A002~A005) |
| 관련 문서 | IA v0.5 화면구조(영상 생성), 기능명세서 v0.5 S6, 프롬프트가이드 v4 S4, 마일스톤 v3.2 S7(비용 모니터링 규칙) |

**기본 흐름**
1. 사용자가 영상 생성 화면에 3x3 그리드 이미지(9개 컷)를 업로드한다
2. 영상 프롬프트를 입력한다 (Shot별 장면 묘사 — 프롬프트가이드 v4 S4 기준)
3. 모델을 선택한다 — 기본값 **Kling 3.0**(개발 테스트, 저비용), 최종 데모·고품질 확인 시 **Seedance 2.0**
4. 영상 길이(5초 기본)를 선택한다
5. [영상 생성] 클릭
6. 시스템이 요청을 접수하고 상태를 "대기 중 → 생성 중"으로 표시한다 (5초 간격 폴링) — **예상 소요 시간은 모델·길이에 따라 수십 초~수 분** 단위로 이미지보다 훨씬 길기 때문에, "대기 중"/"생성 중" 단계 구분과 진행 표시(가짜 프로그레스 바 0→80%)로 사용자가 멈춘 게 아니라는 걸 명확히 알려준다
7. 생성 완료 시 인라인 비디오 플레이어로 결과가 표시된다
8. 사용자가 [저장]으로 라이브러리에 보관한다

**모델 선택 근거** (Kling 3.0 vs Seedance 2.0)

| 구분 | Kling 3.0 (기본값) | Seedance 2.0 |
|------|------|------|
| 비용 | 저비용 — 개발 테스트 기본 | Kling 대비 훨씬 비쌈 |
| 용도 | 반복 테스트, 대사·립싱크가 필요한 씬 | 최종 데모·고품질 확인용 |
| 프롬프트 구조 표준 | Scene→Characters→Action→Camera→Audio&Style | Style/Duration 헤더+Shot 라벨 (본 서비스 프롬프트 구조의 기준 모델) |
| 전환 전략 | — | 안정화 확인 후 데모 직전에만 전환 (비용 통제, 마일스톤 S7 비용 모니터링 규칙) |

**대안/예외 흐름**
- 프롬프트가 빈값이면 [영상 생성] 비활성화
- 10분 초과 시 타임아웃 에러 + [다시 시도] (새 요청으로 처음부터 재시작)
- 폴링 응답이 `ERROR` 상태면 에러 메시지 + [다시 시도]
- 화면 이탈 시 폴링은 중단되지만 서버 측 생성은 계속 진행 (완료분의 라이브러리 반영 여부는 BE 저장 시점 설계에 따름)

1. 사용자가 결과 영역에서 [재생성] 클릭
2. 기존 프롬프트·모델·길이 값이 채워진 입력 화면으로 돌아간다
3. 사용자가 필요한 부분만 수정한다 (또는 그대로 재요청)

**대안/예외 흐름**

## UC-REVERSE-01. 역프롬프트 — 업로드→분석→수정→재생성

| 항목 | 내용 |
|------|------|
| 유즈케이스명 | 역프롬프트 — 업로드→분석→수정→재생성 |
| 유즈케이스 ID | UC-REVERSE-01 |
| 주 액터 | 방송 PD |
| 보조 액터 | Claude Sonnet 멀티모달 분석 엔진, GPT Image 2 API, AWS 스토리지 |
| 목표 | 마음에 드는 레퍼런스 이미지 하나로, 비슷한 스타일의 새 이미지를 빠르게 재현한다 |
| Preconditions | 로그인 상태일 것 / 역프롬프트 화면에 진입해 있을 것 / 참고할 레퍼런스 이미지 파일을 보유하고 있을 것(jpg/png/webp, 10MB 이하) |
| Postconditions | 레퍼런스 이미지 기반으로 재생성된 새 이미지가 라이브러리에 저장된 상태 — 목표는 업로드부터 재생성까지 5분 이내 완료 |
| 우선순위 | P2 (S등급 도전 과제) |
| 관련 문서 | 기능명세서 v0.5 S8, 마일스톤 v3.2 S3(등급 조건) |

**기본 흐름**
1. 사용자가 레퍼런스 이미지를 업로드한다 (드래그앤드롭 또는 파일 선택)
2. [프롬프트 추출] 클릭
3. 시스템(Claude Sonnet 멀티모달)이 이미지를 분석해 스타일·조명·구도·렌즈·색감·분위기 6개 항목과 영문 프롬프트(`prompt_en`)를 추출한다
4. 추출 결과가 태그 칩 형태로, 프롬프트는 에디터에 표시된다
5. 사용자가 프롬프트를 직접 수정한다 (원본 보기 토글로 추출 원본과 비교 가능)
6. [이미지 생성으로 이어서] 클릭
7. 수정된 프롬프트가 이미지 생성 화면(메뉴 A/B)에 자동으로 채워지고, 이후 UC-IMAGE-01/UC-IMAGE-02 흐름을 그대로 따른다
8. 생성된 이미지를 저장하면 라이브러리에 반영된다

**대안/예외 흐름**
- 미지원 형식·10MB 초과 업로드 시 각각 에러 메시지 표시, 업로드 진행 불가
- 분석 실패 시 에러 메시지 + [다시 시도] (업로드 파일은 유지)
- [원본으로 되돌리기]로 수정 내용을 추출 시점으로 되돌릴 수 있다
