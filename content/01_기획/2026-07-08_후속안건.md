## 안건 1. 메뉴 B 키이미지 — 상세 프롬프트 입력 방식

**배경**
키이미지 생성(메뉴 B)은 캐릭터 설정값(헤어·의상·액세서리·컬러 코드 등)이 포함된 상세 프롬프트가 필요하다. 그러나 일반 사용자가 이를 직접 작성하기는 어렵다.

**검토 중인 방식**

| 방식 | 설명 | 장점 | 단점 |
|------|------|------|------|
| 텍스트 직접 입력 | 사용자가 직접 상세 프롬프트 작성 | 자유도 높음 | 비전문 사용자 진입 장벽 높음 |
| 속성 선택 사이드바 | 성별·헤어·의상·컬러 등 항목 선택 → 자동 프롬프트 생성 | 누구나 쓸 수 있음 | 속성 항목 설계 필요, 개발 공수 증가 |
| 혼합 (직접 입력 + 속성 선택) | 두 가지 모두 제공 | 유연함 | 가장 복잡 |

**논의 필요 사항**
- 속성 선택 사이드바 구현 시 BE 스키마 변경 필요 여부
- Claude Sonnet을 "캐릭터 설정 → 최적화 프롬프트 자동 변환" 레이어로 활용하는 방안 검토 (→ 프롬프트 가이드 v2 참고)

**담당**: PM + BE + UX/UI 협의 필요
**우선순위**: 중 (메뉴 B는 A001 기능, A등급 착수 시점에 결정)

## 안건 2. 메뉴 A·B 결과물 라이브러리 저장 구조

**배경**
현재 라이브러리(A006)는 이미지·영상을 저장하는 구조로 설계되어 있다. 메뉴 B(키이미지)는 단순 이미지 외에 캐릭터 설정 데이터(컬러 코드, 의상 정보 등)를 함께 저장할 필요가 있을 수 있다.

**검토 필요 항목**
- 이미지 파일만 저장할지, 캐릭터 설정 데이터도 함께 저장할지
- 저장 시 BE DB 스키마 변경 범위
- 라이브러리 화면에서 메뉴 A 결과물(배경·캐릭터 다각도)과 메뉴 B 결과물(캐릭터 시트)을 구분해서 표시할지 여부
- 현재 라이브러리 방향(히스토리) vs 차후 방향(좋아요·스크랩) 전환 시 구조 영향

**담당**: BE 검토 후 PM·UX/UI 협의
**우선순위**: 중 (A006 구현 전 결정 필요)

## 안건 3. Claude Sonnet 한국어 프롬프트 변환 레이어 — 개발 일정 조율

**배경**
GPT Image 2는 영어 프롬프트에 최적화되어 있어, 한국어 입력 시 Claude Sonnet을 통해 최적화된 영어 프롬프트로 변환하는 방안이 논의됐다. (→ 프롬프트 가이드 v2 참고)

**현재 상황**
- 기술적 방향은 확인됨
- M001~M002(이미지 생성 API 호출)에 Claude Sonnet 변환 단계 추가 시 BE API 호출 구조 변경 필요
- 2주차 Pass 범위(M001~M006)가 이미 설계된 상태에서 변경 시 개발 부담

**논의 필요 사항**
- BE에서 Claude Sonnet 변환 레이어 추가 가능 시점
- 레이턴시 영향 (Claude 변환 1~3초 + GPT Image 2 생성 10~30초)
- 기능명세서 M001~M002 업데이트 시점

**담당**: BE 검토 필요 (PM 제안 후 협의)
**우선순위**: 하 (M001~M006 완성 후 논의)

## 안건 4. 로그인 통합 — B1팀 요청 대응

**배경**
B1팀 고다희 PM이 전체 PM 채널에 사내 로그인 및 user 테이블 통합에 대한 의견 조율을 요청했다. (2026-07-07)

**B1팀 방향**
- 사내 보안 가이드라인에 따라 자유 회원가입 막음
- 허가된 사내 인원(화이트리스트)만 구글 소셜 로그인(OAuth)으로 진입

**A2팀 현재 방향**
- 기본 이메일·비밀번호 로그인 (소셜 로그인 없음)
- 자유 가입 허용

**확인 필요 사항**
1. 사내 보안 가이드라인이 A2팀에도 동일하게 적용되는지 (PD님 확인 필요)
2. 전체 프로젝트 통합 시 user 테이블 공통 컬럼 기준 (`user_id`, `team`, `role` 등)
3. A2팀 로그인 방식 변경 필요 여부 및 시점
4. A2팀 user 테이블 초안 현황 (BE 담당자 확인 필요)

**담당**: PM → PD님 확인 / PM + BE 협의
**우선순위**: 상 (B1팀 회신 요청 — 가능한 빨리)

**업데이트 필요 항목**

**담당**: PM (안건 1·2 결정 후 착수)
**우선순위**: 중 (안건 1 결정 후)
