## 1. 서비스 개요

### 1.1 서비스 한 줄 정의

> **방송 제작자가 텍스트 기획 하나로 이미지·영상을 생성하고, 레퍼런스를 역분석해 프롬프트를 추출하는 AI 생성 엔진**

### 1.2 서비스명

| 구분 | 내용 |
|------|------|
| 임시명 | KX GenStudio |
| 확정명 | 미정 (팀 논의 후 결정) |

### 1.3 배경 및 목적

방송 제작 현장에서 PD·AD는 기획 단계부터 시각화 작업에 많은 시간을 소모한다. 현재는 Midjourney, Runway, ElevenLabs 등 여러 사이트를 번갈아 사용해야 해 워크플로우가 파편화되어 있다.

KX GenStudio는 이미지 생성·영상 생성·역프롬프트 기능을 한 화면에서 제공해 방송 제작진의 사전 시각화 작업 시간을 단축한다. 스토리보드 빌더는 B팀 서비스에서 담당한다.

### 1.4 서비스 포지셔닝

```
                    고품질
                      │
        Magnific      │      GPT Image 2
        (업스케일)     │      (단일 이미지)
                      │
일반 크리에이터 ───────┼─────── 방송 제작자
타겟                  │              타겟
                      │
        Krea          │    ★ KX GenStudio
        (실시간)       │    (생성+영상+역프롬프트
                      │     통합 / 방송 특화)
                   저비용
```

> 우리의 차별점: 여러 AI 서비스를 오가지 않아도 되는 **방송 제작 특화 통합 워크플로우**

## 2. 타겟 사용자

### 2.1 주요 타겟

| 구분 | 내용 |
|------|------|
| 직군 | 방송 PD, AD, 영상 기획자 |
| 특징 | AI에 익숙하지 않을 수 있음, 프롬프트 작성 어려움 |
| 핵심 니즈 | 복잡한 프롬프트 없이도 원하는 이미지·영상을 빠르게 뽑고 싶다 |

### 2.2 페르소나

**페르소나 A — 기획 PD 김○○ (35세)**
- 드라마 프리 프로덕션 단계에서 스토리보드 시각화가 필요
- AI 툴 경험 없음, 한국어로 기획서를 작성하고 그걸 바로 이미지로 보고 싶음
- 레퍼런스 영화의 장면 느낌을 그대로 재현하고 싶은데 프롬프트를 모름
- **KX GenStudio 사용 이유:** 한국어 기획 텍스트 → 이미지, 레퍼런스 영상 → 프롬프트 추출

**페르소나 B — 조연출 AD 이○○ (28세)**
- 촬영 전 감독에게 9컷 스토리보드를 빠르게 만들어 공유해야 함
- Figma는 쓸 줄 알지만 AI 이미지 생성은 처음
- 매번 Midjourney에 영어 프롬프트 치는 게 번거로움
- **KX GenStudio 사용 이유:** 기획 텍스트 → 9컷 그리드 이미지 한 장으로 빠르게 생성

## 3. 핵심 기능 및 우선순위

| ID | 기능명 | 입력 | 출력 | 사용 API | 우선순위 |
|----|--------|------|------|---------|---------|
| F001 | 이미지 생성 | 텍스트 프롬프트 | 키이미지 | GPT Image 2 / Gemini (테스트) | **필수** |
| F003 | 영상 생성 | 텍스트 프롬프트 또는 키이미지 | 영상 클립 | Seedance 2.0 / Kling 3.0 | **필수** |
| F004 | 역프롬프트 | 레퍼런스 이미지 | 재현용 프롬프트 | Claude Sonnet | **권장** |
| F005 | 미디어 라이브러리 | — | 생성물 저장·조회 | Supabase | **필수** |
| F006 | 다운로드 | — | 이미지·영상 파일 | — | 선택 |
| F007 | 라이브러리 검색·필터 | — | 태그·날짜 정렬 | — | 선택 |

### 3.1 기능 간 의존 관계

```
F001 이미지 생성
    └─ 생성물 → F005 라이브러리 저장
    └─ 사용자가 로컬 저장 후 B팀 서비스에 직접 업로드 (A-B 수동 연동)

F003 영상 생성 (비동기)
    └─ fal.ai queue submit → polling → result
    └─ 생성물 → F005 라이브러리 저장

F004 역프롬프트
    └─ Claude Sonnet 멀티모달로 이미지 분석 → 프롬프트 추출
    └─ 추출된 프롬프트 → F001 이미지 생성으로 재활용 가능
```

## 4. 주요 사용자 시나리오

### 시나리오 1 — 레퍼런스 영상의 분위기를 내 작업에 재현하기 (F004 + F001)

> **배경:** PD 김○○은 좋아하는 영화의 색감과 조명을 드라마 키이미지에 적용하고 싶다.

```
1. [역프롬프트] 탭 선택
2. 레퍼런스 이미지 또는 영상 업로드
3. [프롬프트 추출] 버튼 클릭
4. Claude Sonnet이 스타일·조명·구도·렌즈 키워드 포함 영문 프롬프트 추출
5. 추출된 프롬프트 복사 또는 [이미지 생성으로 이어서] 버튼 클릭
6. [이미지 생성] 탭에서 프롬프트 수정 후 키이미지 생성
7. 원하는 분위기가 재현됐는지 Before/After로 비교
8. 만족하면 라이브러리 저장
```

**성공 기준:** 레퍼런스 이미지 업로드 → 프롬프트 추출 → 재생성까지 5분 이내

---

### 시나리오 2 — 키이미지 기반으로 영상 클립 생성하기 (F003)

> **배경:** 이미지 생성으로 만든 키이미지를 바탕으로 영상 클립을 뽑고 싶다.

```
1. [영상 생성] 탭 선택
2. 프롬프트 직접 입력 또는 키이미지를 첫 프레임으로 업로드
   (B팀 서비스에서 스토리보드 작업 후 받은 프롬프트 활용 가능)
3. 영상 길이 선택 (5초 / 10초 / 15초)
4. [영상 생성] 버튼 클릭
5. 비동기 처리 시작 → 진행 상태 표시 (생성 중... / 완료)
6. 완료 후 영상 미리보기 + 라이브러리 저장
7. 필요 시 다운로드
```

**성공 기준:** 영상 생성 요청 후 상태 표시 정상 동작, 결과물 라이브러리 저장까지 완료

## 6. MVP 범위 정의

### Must Have (4주 안에 반드시)
- [ ] F001 이미지 생성 (프롬프트 → 이미지, 모델 선택)
- [ ] F003 영상 생성 (프롬프트·키이미지 → 영상 클립, 비동기 처리)
- [ ] F005 미디어 라이브러리 (저장·조회)
- [ ] 이미지·영상 다운로드 기본 기능

### Should Have (여유 있으면)
- [ ] F004 역프롬프트 (이미지·영상 → 프롬프트 추출)
- [ ] 역프롬프트 결과 → 이미지 생성으로 바로 연결
- [ ] 라이브러리 필터 (타입·날짜)

### Could Have (도전 과제)
- [ ] 라이브러리 검색 (태그·키워드)
- [ ] 영상 클립 이어붙이기 (ffmpeg 자동 stitching)
- [ ] 배치 생성 (여러 장 동시)
- [ ] 프롬프트 히스토리·즐겨찾기

### Out of Scope (이번 MVP 제외)
- 올인원 통합 플랫폼 (B·C팀 기능 통합) → 고도화 단계
- 실시간 렌더링 (Krea 방식)
- 캐릭터 일관성 보장 시스템 (Soul ID 수준)
- 영상 편집 기능
- 소셜 공유·팀 협업 기능

## 7. 기술 스택

| 구분 | 기술 | 용도 |
|------|------|------|
| 프레임워크 | Next.js 14+ (App Router) | 풀스택 기반 |

| 언어 | TypeScript | 타입 안정성 |

### API 전략

```
이미지 개발·테스트   →  Gemini 2.5 Flash (빠름·저렴)
이미지 데모·최종     →  GPT Image 2 (품질 우선)
역프롬프트           →  Claude Sonnet (멀티모달 이미지 분석)
```

## 8. 평가 기준

### 중간점검 (2주차) — Pass 기준

| 항목 | Pass 기준 |
|------|---------|
| 기획 완성도 | 핵심 기능·화면 설계 포함 |
| API 연동 | 이미지 생성 호출 성공 |
| 핵심 기능 | 프롬프트 → 이미지 → 라이브러리 저장 동작 |

### 최종 평가 (4주차) — 등급 기준

| 등급 | 기준 |
|------|------|
| S | 이미지+영상+역프롬프트+라이브러리+배포 |
| A | 이미지+영상+라이브러리+배포 |
| B | 이미지+라이브러리+배포 |
| C | 이미지 생성만 동작 |
| Fail | API 연동 실패 |

**클라이언트(PD) 평가 포인트**
- 이미지 생성 결과물이 방송 제작 현장에서 레퍼런스로 쓸 만한 퀄리티인가
- 영상 비동기 처리 매끄러움 (로딩 상태 표시)
- 역프롬프트 정확도 (레퍼런스 재현성)
- 방송 제작자가 AI 지식 없이도 직관적으로 쓸 수 있는가
