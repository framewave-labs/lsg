## 1. 프로젝트 구조 확정 (전체 공통)

### 올인원 → 개별 프로젝트로 변경 확정
원래 PD님 원안은 모든 기능을 한 페이지에서 통합하는 올인원 플랫폼이었으나, 아래 이유로 **각 팀이 독립 프로젝트로 개발**하는 것으로 합의됐습니다.

- 기간 1개월, 가용 시간 주 15시간(전체 ~60시간), 인턴 수준
- 처음부터 통합하면 코드가 방대해져 합치기 어려움
- 포트폴리오용으로도 개별이 유리

→ C팀은 **오디오 툴킷** 프로젝트를 독립적으로 완주하시면 됩니다.

### 전체 워크플로우 내 C팀 위치

```
기획(텍스트)
    → A팀: 스토리보드 이미지 생성
    → B팀: 스토리보드 텍스트 추출
    → A팀: 영상 생성 + 음성(TTS) 삽입  ← 음성까지 A팀에서 처리
    → C팀: 음성이 붙은 영상 분석 → 효과음 자동 삽입 + 자막 생성  ← 여기
```

> **중요:** A팀이 음성까지 입힌 영상을 C팀에 넘기는 구조입니다.
> C팀은 해당 영상을 분석해 효과음을 적재적소에 삽입하고, 자막을 생성·편집·출력하는 역할입니다.

### UI/UX 디자인 시스템 통일 (중요)
나중에 통합할 여지를 남기기 위해 **폰트·컬러·디자인셋은 전 팀이 동일하게** 가져갑니다.
- 레퍼런스 공유 및 최종 결정은 **PD님**이 하실 예정
- 각 팀 UX/UI 담당자가 PD님과 직접 조율

## 2. C팀 프로젝트 개요

### 서비스 소개
> 영상을 분석해 효과음을 자동 삽입하고, 음성을 받아써서 자막으로 만든 뒤 맞춤법 검사·수정·SRT 출력까지 한 화면에서 처리하는 오디오 후반작업 도구

두 가지 기능으로 구성됩니다:
1. **SFX 탭:** 영상을 업로드하면 영상을 분석해 효과음을 자동으로 적재적소에 삽입
2. **자막 탭:** 영상·음성 파일을 업로드하면 STT로 받아쓰기 → 맞춤법 검사 자동 실행 → 인라인 수정 → SRT 파일 다운로드

## 3. PD님 추가 요청사항 (상세)

### 3.1 자막 탭

**현재 현업 워크플로우:**
```
STT로 텍스트 추출 → SRT 파일 생성 → 프리미어에 올림 → 텍스트 확인·수정
```

**PD님이 원하는 C팀 서비스 워크플로우:**
```
영상·음성 업로드
    → STT 자동 받아쓰기
    → 맞춤법 검사 자동 실행
    → 화면에서 직접 텍스트 수정
    → SRT 파일로 다운로드
    → 프리미어에 올려서 바로 사용
```

**핵심 포인트:**
- STT 결과에 맞춤법 오류가 있을 수 있어 **맞춤법 검사기 자동 실행** 필수
- 수정을 위해 별도 툴로 이동하지 않고 **서비스 화면 내에서 바로 편집** 가능해야 함
- 수정 완료 후 **SRT 확장자로 바로 다운로드** 가능해야 함

### 3.2 SFX 탭

**PD님이 원하는 기능:**
- 편집 완료된 영상을 C팀 서비스에 업로드
- 영상을 분석해 **효과음이 필요한 구간을 자동으로 파악**
- 사내 효과음 라이브러리 또는 생성 API를 활용해 **적재적소에 효과음 자동 삽입**
- 업로드한 **영상 길이와 상관없이 전체 영상 그대로 처리** 가능해야 함

**기술 구현 이슈 — 영상 길이 제한:**

현재 Kling 등 AI 영상 분석·생성 모델은 처리 가능한 영상 길이에 제한이 있습니다.
- Kling: 최대 **20초** 단위 처리
- 현업에서는 이를 우회하기 위해 **20초 단위로 잘라서 처리 후 이어붙이는 방식** 사용 중

> **→ 구현 방향 제안 (팀과 논의 필요):**
> 긴 영상을 자동으로 일정 단위(20초 등)로 분할 → 각 구간별 효과음 처리 → 다시 합산
> 이 파이프라인을 C팀 서비스 내에서 자동화하면 PD님 요청(길이 무관 처리)에 근접 가능

## 4. 효과음 라이브러리 현황

- 회사 내 **방대한 양의 효과음 보유** 중
- 단, 각 PD마다 스타일이 다르고 효과음을 별도로 보유하고 있어 **사내 규격화가 되어 있지 않은 상태**
- 파일명·카테고리 통일이 안 되어 있어 **아카이빙 작업 필요**
- **PD님께서 카테고리 1차 분류(폴더 정리) 후 C팀에 전달** 예정
- C팀은 전달받은 라이브러리를 서비스에 연동해 효과음 자동 삽입 시 활용

## 5. 기술 스택

| 구분 | 기술 | 용도 |
|------|------|------|
| 프레임워크 | Next.js 14+ | 풀스택 |
| 스타일링 | Tailwind CSS | UI |
| **SFX 분석·삽입** | **영상 분석 API (검토 필요)** | 영상 → 효과음 자동 삽입 |
| **STT (자막)** | **OpenAI Whisper** | 음성→텍스트+타임스탬프 |
| STT 대안 | AssemblyAI | 단어 단위 타임스탬프 (정밀도 높음) |
| 맞춤법 검사 | 한국어 맞춤법 API (검토 필요) | 자동 맞춤법 교정 |
| 자막 생성 | pysubs2 / srt | SRT·ASS 포맷 생성 |
| 영상 분할·합산 | ffmpeg | 긴 영상 처리 파이프라인 |
| 배포 | Vercel | 호스팅 |

> **검토 필요 항목:**
> - 영상 분석 후 효과음 자동 삽입 API → 어떤 API 사용할지 BE와 논의 필요
> - 한국어 맞춤법 검사 API → 부산대 맞춤법 검사기, 네이버 맞춤법 검사기 등 검토 필요
