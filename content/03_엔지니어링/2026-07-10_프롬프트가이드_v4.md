## 1. 한국어 프롬프트 처리 방식

### 1.1 배경

GPT Image 2는 영어 프롬프트에 최적화되어 있다. 그러나 A2팀 서비스 사용자는 방송 제작자로, 한국어로 기획 의도를 입력하는 것이 훨씬 자연스럽다. 이 간극을 메우기 위해 **Claude Sonnet을 프롬프트 변환 레이어로 활용**한다.

### 1.2 처리 흐름

```
사용자 (한국어 입력)
    ↓
Claude Sonnet
  · 한국어 → 영어 변환
  · 구도·조명·분위기·스타일 등 시각 요소 구체화
  · GPT Image 2 최적화 포맷 적용 (80단어 이내, photorealistic 끝맺음)
    ↓
GPT Image 2 (최적화된 영어 프롬프트로 생성)
    ↓
이미지 결과
```

**단순 번역이 아니라 "최적화 변환"이 핵심이다.**

| 사용자 입력 (한국어) | Claude 변환 결과 (영어) |
|-------------------|----------------------|
| 빗속에서 우산 들고 걷는 주인공 | `the protagonist walking through a narrow alley in heavy rain, holding an umbrella, medium shot, dramatic side lighting reflecting off wet pavement, cold blue tones, tense mood, Korean drama style, cinematic, photorealistic, high detail` |
| 황혼 무렵 한강변을 걷는 노부부 | `elderly couple walking along the Han River at dusk, holding hands, wide shot, warm golden hour lighting, silhouette against orange sky, peaceful and nostalgic mood, cinematic, photorealistic, high detail` |

### 1.3 UX 방식 (반자동 — 권장)

사용자가 한국어로 입력하면 Claude가 변환한 **영어 프롬프트를 FE에 노출**하고, 사용자가 수정 후 생성하는 방식을 권장한다.

```
[한국어 입력창]
  "빗속에서 우산 들고 걷는 주인공"

  ↓ 변환됨 (Claude Sonnet)

[영어 프롬프트 미리보기 · 수정 가능]
  "the protagonist walking through a narrow alley..."

  → [이미지 생성] 버튼
```

**이 방식을 권장하는 이유**
- 사용자가 변환 결과를 확인하고 수정할 수 있어 생성 품질 통제 가능
- 역프롬프트(S등급)의 `prompt_en` 노출 방식과 UX 일관성 유지
- 사용자가 자연스럽게 영어 프롬프트 패턴을 학습하게 됨

### 1.4 Claude Sonnet 시스템 프롬프트 초안 (이미지)

```
You are a professional prompt engineer specializing in AI image generation.

When given a Korean description, convert it into an optimized English prompt for GPT Image 2.

Rules:
- Keep the original intent and subject intact
- Add specific visual elements: shot type, camera angle, lighting, color tone, mood, style
- End every prompt with: photorealistic, high detail
- Keep under 80 words
- Output only the English prompt — no explanation, no Korean

Example:
Input: 빗속에서 우산 들고 걷는 주인공
Output: the protagonist walking through a narrow alley in heavy rain, holding an umbrella, medium shot, dramatic side lighting reflecting off wet pavement, cold blue tones, tense mood, Korean drama style, cinematic, photorealistic, high detail
```

> 이 시스템 프롬프트는 BE 구현 시 조정 가능. 현재는 초안. 영상용 시스템 프롬프트는 4.4 참고.

## 2. 이미지 프롬프트 작성 원칙

### 2.1 기본 구조 (순서 중요)

```
[피사체 + 동작]  →  [구도/샷 타입]  →  [조명]  →  [분위기/색감]  →  [스타일]  →  [품질 지시어]
```

**예시:**
```
A woman in a hanbok standing in a palace courtyard,    ← 피사체
wide shot, slightly high angle,                        ← 구도
golden hour sunlight, soft shadows,                    ← 조명
warm amber tones, nostalgic mood,                      ← 분위기/색감
cinematic, Korean period drama style,                  ← 스타일
photorealistic, high detail, 4K                        ← 품질
```

### 2.2 필수 끝맺음 (모든 프롬프트에 포함)

```
photorealistic, high detail
```
→ GPT Image 2는 이 두 단어가 있을 때 결과물 품질이 현저히 올라갑니다.

### 2.3 피해야 할 것

| 하지 말 것 | 이유 |
|-----------|------|
| 한국어로 직접 GPT Image 2에 입력 | Claude Sonnet 변환 레이어를 반드시 거칠 것 |
| 너무 긴 프롬프트 (80단어 초과) | 후반부 키워드 반영률 하락 |
| 추상적 감정만 적기 ("sad" 만) | 구체적 시각 요소로 변환 필요 ("tears on cheek, drooping shoulders") |
| 유명인 이름 사용 | API 정책 위반 가능 |
| 텍스트/글씨 요청 | 이미지 내 텍스트 품질이 불안정함 |

## 4. 영상 프롬프트 작성 원칙

### 4.1 입력 구조 개요

영상 생성 입력은 **3x3 스토리보드 그리드 이미지 1장 + scene(=shot) 1~9 프롬프트**다. 9개 shot은 각각 독립된 장면 설명이지만, 최종 영상 요청 시에는 하나의 영상 맥락으로 함께 전달된다.

- **기본 모델은 Seedance 2.0** — 아래 4.2가 실제 사용 기준
- **Kling 3.0은 테스트/저비용 대안** — 4.3에서 다룸. 대사·오디오가 중요한 장면이나 빠른 프로토타입 검증에 적합
- 3x3 그리드 순서는 좌상단부터 우하단까지 scene 1~9
- 5초 이하의 단순 테스트 클립은 단일 shot(주요 동작 1가지)만으로 검증 가능 — 아래 7장 테스트 세트는 이 방식을 사용

### 4.2 Seedance 2.0 최적화 (기본 모델)

**핵심 원칙**
Seedance는 이미지 생성용 키워드("아름다운 조명, 4K")에는 반응하지 않는다. **영화 촬영 지시서**처럼, 무엇이 움직이고 카메라가 어떻게 움직이는지를 구체적으로 써야 한다.

| 순서 | 항목 | 설명 |
|---|---|---|
| 1 | 주체·액션 | 무엇이 일어나는가를 가장 먼저 쓴다 |
| 2 | 카메라 움직임 | dolly, tracking shot, handheld, POV, aerial shot, rack focus 등 실제 촬영 용어를 쓴다 — 모델이 이를 문체가 아니라 촬영 지시로 해석한다 |
| 3 | 사운드 | "천둥소리", "양철 지붕 위 빗소리"처럼 구체적인 음향을 명시한다. 대사·립싱크는 아직 정확도가 낮아 다른 오디오 요소를 우선한다 |
| 4 | 장면 전환 | 멀티샷일 때 각 샷이 어떻게 이어지는지 |

**한 샷당 원칙**: 주요 액션 1개 + 카메라 움직임 1개만 담는다. 과부하를 주면 모델이 임의로 우선순위를 골라버린다.

**표준 형식** (`Shot` 라벨 필수 — "Scene"이 아니라 "Shot"이 모델이 인식하는 라벨)

```
Style: [전체 스타일 요약 — 장르·미학·톤]
Duration: [총 길이]s

[MM:SS-MM:SS] Shot 1: [프레이밍 제목].
Scene: [장소/상황]
Action: [행동 묘사]
Sound: [구체적 음향]
[선택: Lighting / Dialogue Cue / Detail / Emotional Performance 등 필요한 라벨만 추가]

[MM:SS-MM:SS] Shot 2: ...
```

상단에 `Style`·`Duration`을 한 번만 선언하고, 각 shot은 타임코드(`[00:00-00:02]`)로 구간을 표시한 뒤 필요한 라벨만 붙인다. fps 태그는 쓰지 않는다 — `Duration: Xs` + 타임코드로 길이·구간을 표현한다.

**프레이밍 표기**: 서술형 괄호("(Interior/Close-up)", "(Wide Action)")와 약어(ELS/ECU/MCU/OTS 등) 둘 다 사용 가능.

| 표기 | 뜻 |
|---|---|
| ELS | Extreme Long Shot |
| ECU | Extreme Close-Up |
| MCU | Medium Close-Up |
| OTS | Over-The-Shoulder |
| Two Shot | 두 인물의 관계 표현 |

**작성 기준**

| 항목 | 기준 |
|---|---|
| 길이 | 단일 샷 2~4문장 권장(멀티샷 맥락에서는 1~2문장으로 압축) |
| 사운드 | 장면마다 구체적 음향 최소 1개는 명시 — 없으면 제네릭한 배경음으로 채워짐 |
| 움직임 | `a person walking` (X) → `a woman slowly walks, slightly hunched shoulders` (O) |
| 정적 구도 | 카메라를 안 움직일 거면 "static tripod", "fixed-camera"로 명시 — 지시가 없으면 모델이 임의로 움직임을 넣을 수 있음 |
| 텍스트·로고 제약 | 읽을 수 있는 텍스트·로고·숫자가 나오지 않게 할 경우 "no readable text, logos, or numbers" 문구 사용 |
| 일관성 | 인물·장소·시간대·색감이 갑자기 바뀌지 않게 유지 |
| 길이 옵션 | Seedance 2.0: 4~15초. 멀티샷 구성일 때는 Auto(모델이 자동 배분) 권장 |

**자주 하는 실수**

| 하지 말 것 | 이유 |
|---|---|
| "Shot 1:" 같은 라벨 생략 | 여러 장면이 하나의 연속 샷으로 뭉쳐서 생성됨 |
| 한 샷에 액션·카메라·분위기 변화 동시 압축 | 모델이 임의로 우선순위를 골라버림 |
| 사운드 묘사 생략 | 제네릭한 배경음으로 채워짐 |
| "좋은 앵글로" 같은 막연한 카메라 지시 | 무엇을·어디서 비추는지 구체적으로 써야 함 |
| 카메라 스펙(mm, fps, ISO) 나열 | Seedance는 기술 스펙보다 촬영 지시·리듬 언어에 더 잘 반응 |

### 4.3 Kling 3.0 최적화 (보조/테스트용)

Seedance 2.0의 대사·립싱크 정확도가 아직 낮기 때문에, **대사가 중요한 씬이나 빠른 저비용 프로토타입 검증**에는 Kling 3.0을 사용한다.

**구조 원칙**: "감독의 지시문"처럼 읽히는 프롬프트에 가장 잘 반응. 권장 순서:
```
Scene → Characters → Action → Camera → Audio & Style
```

**멀티샷**: 최대 **6샷**을 하나의 출력에 담을 수 있음 — Seedance의 9-shot 그리드보다 적으므로, 짧은 시퀀스 검증용으로 적합.

**캐릭터 일관성**: 동일 캐릭터를 여러 샷에 쓸 경우 레퍼런스 이미지를 고정하고, 캐릭터 묘사 문구를 **토씨 하나 안 바꾸고 반복**한다. 메뉴 B 캐릭터 시트(예: "미라 한" — [[2026-07-08_메뉴AB_프롬프트가이드_v1]] 3.3)의 SUBJECT/CLOTHING/ACCESSORIES 필드 문구를 그대로 복붙하는 방식을 권장.

**네이티브 오디오·대사**: 대사·환경음·음성 톤 지정을 프롬프트 안에서 지원한다. 오디오를 쓸 경우 **누가, 언제 말하는지** 명시(멀티 캐릭터 씬에서 필수).
```
Audio: [캐릭터명]이 "[대사]"라고 말함, [톤/감정] / 배경음: [환경음 종류]
```

**손동작 앵커링**: "she moves her hands" (X) → "her fingers firmly grip the edge of the ceramic cup" (O) — 손동작을 허공에서 처리하면 어색하게 렌더링됨.

### 4.4 엔진 선택 및 한국어 입력 처리

- 한국어 입력 → 영어 변환은 1.2의 **Claude Sonnet 변환 레이어**를 그대로 사용하되, 시스템 프롬프트에 **엔진 분기**를 추가한다 (기본값 Seedance 2.0 / 대사·프로토타입 검증 시 Kling 3.0 선택)
- FE 노출 방식도 1.3(반자동 — 변환 결과 미리보기 후 수정 가능)과 동일하게 유지 권장

**엔진별 변환 시스템 프롬프트 분기 예시**
```
[공통] 사용자 의도·피사체를 유지하며 시각 요소(구도/조명/분위기)를 구체화

[Seedance 2.0 — 기본값]
- 상단에 Style/Duration 헤더 선언
- 각 shot을 "Shot N:" 라벨 + 타임코드로 구분
- Scene/Action/Sound 라벨 기본, 필요 시 Lighting/Dialogue Cue/Detail 추가
- 한 샷당 액션 1개 + 카메라 움직임 1개만

[Kling 3.0 — 대사·프로토타입용]
- Scene → Characters → Action → Camera → Audio & Style 순서로 출력
- 멀티샷일 경우 SHOT 1/2/3... 라벨링 (최대 6샷)
- 손동작은 반드시 사물에 앵커링
```

> 이 분기 로직은 BE 구현 시 조정 가능. 현재는 초안.

### 7.1 멀티샷(9-shot) 워크드 예시 — "미라 한" 등장 씬

> 위 20개는 단일 shot 검증용이고, 실제 서비스는 9-shot 그리드이므로 멀티샷 완성형도 1개 참고용으로 남긴다. 메뉴 B에서 확정된 "미라 한" 캐릭터([[2026-07-08_메뉴AB_프롬프트가이드_v1]] 3.3)를 예시로 사용. **실제 시연 스크립트가 확정되면 이 예시를 시연 장면으로 교체할 것.**

```
Style: Urban fantasy Korean drama, anime style, warm-to-cool tonal shift, dominant black #1E1E24 and vibrant purple #7B61FF accents.

Action: 미라 한이 옥상 난간 쪽으로 천천히 걸어감, slow push-in.

[05-09s] Shot 3: City Glow (Slow Dolly Out).
```
