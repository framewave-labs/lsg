## 1. 이미지 프롬프트 작성 원칙

### 1.1 기본 구조 (순서 중요)

```
[피사체 + 동작]  →  [구도/샷 타입]  →  [조명]  →  [분위기/색감]  →  [스타일]  →  [품질 지시어]
```

**예시:**
```
A woman in a hanbok standing in a palace courtyard,    ← 피사체
wide shot, slightly high angle,                        ← 구도
golden hour sunlight, soft shadows,                    ← 조명
warm amber tones, nostalgic mood,                      ← 분위기/색감
cinematic, Korean period drama style,                  ← 스타일
photorealistic, high detail, 4K                        ← 품질
```

### 1.2 필수 끝맺음 (모든 프롬프트에 포함)

```
photorealistic, high detail
```
→ GPT Image 2는 이 두 단어가 있을 때 결과물 품질이 현저히 올라갑니다.

### 1.3 피해야 할 것

| 하지 말 것 | 이유 |
|-----------|------|
| 한국어 단어 섞기 | GPT Image 2는 영어 프롬프트에 최적화 |
| 너무 긴 프롬프트 (100단어 초과) | 후반부 키워드 반영률 하락 |
| 추상적 감정만 적기 ("sad" 만) | 구체적 시각 요소로 변환 필요 ("tears on cheek, drooping shoulders") |
| 유명인 이름 사용 | API 정책 위반 가능 |
| 텍스트/글씨 요청 | 이미지 내 텍스트 품질이 불안정함 |

## 3. 영상 프롬프트 작성 원칙

### 3.1 기본 구조

```
[피사체 + 동작]  →  [카메라 무브먼트]  →  [배경/환경]  →  [조명/시간대]  →  [분위기]  →  [스타일]
```

**예시:**
```
A woman slowly walks along a riverside path,          ← 피사체 + 동작
camera gently tracking her from behind,               ← 카메라 무브먼트
autumn leaves falling, golden foliage on both sides, ← 배경
warm golden hour sunlight,                            ← 조명
melancholic and peaceful mood,                        ← 분위기
cinematic, Korean drama style                         ← 스타일
```

### 3.2 영상 프롬프트 특이사항

- **동작을 구체적으로 묘사할수록 결과가 좋습니다**
  - 나쁜 예: `a person walking`
  - 좋은 예: `a woman slowly walks, slightly hunched shoulders, looking down at her phone`
- **카메라 무브먼트를 반드시 포함하세요** — 없으면 대부분 정적 샷으로 나옵니다
- **5초 테스트 클립은 단순한 동작 1가지만** — 복잡한 장면은 10초+로

## 4. 영상 키워드 모음

### 4.1 카메라 무브먼트

| 무브먼트 | 영어 키워드 | 효과 |
|---------|-----------|------|
| 정적 | static shot, locked off camera | 안정·집중 |
| 좌우 패닝 | camera pans left/right | 공간 탐색 |
| 상하 틸팅 | camera tilts up/down | 높이 강조 |
| 돌리인 | dolly in, camera pushes in | 긴장감 상승 |
| 돌리아웃 | dolly out, camera pulls back | 고독·이별 |
| 트래킹 | tracking shot, camera follows | 동행·추격 |
| 핸드헬드 | handheld camera, shaky cam | 긴박·현장감 |
| 드론샷 | drone shot, aerial view | 장대함·자유 |
| 크레인 | crane shot, rising camera | 웅장·상승 |
| 원테이크 | single continuous shot | 몰입감 |

### 4.2 씬 묘사 동사 (동작을 구체화할 때)

| 상황 | 키워드 |
|------|-------|
| 걷기 | walks slowly, strides confidently, stumbles, runs |
| 바라보기 | gazes into distance, stares intensely, glances sideways |
| 앉기/서기 | sits down wearily, stands up abruptly, collapses onto chair |
| 손동작 | reaches out, clenches fist, covers face with hands |
| 변화 | transitions from day to night (time-lapse), seasons change |

### 4.3 영상 분위기 조합 예시

| 장르 | 추천 조합 |
|------|---------|
| 멜로 드라마 | soft tracking shot + golden hour + warm tones + slow motion |
| 스릴러 | handheld camera + dramatic side lighting + cool blue + fast cuts feel |
| 예능·버라이어티 | wide static shot + bright daylight + vibrant colors + energetic |
| 다큐멘터리 | handheld + natural light + desaturated + documentary style |

## 6. 이미지 품질 판단 기준

2주차 이미지 생성 완성 시 아래 기준으로 판단합니다.

### Pass 기준 (최소)

| 항목 | 기준 |
|------|------|
| 프롬프트 반영도 | 핵심 피사체·분위기가 이미지에 반영됨 |
| 해상도 | 흐릿하지 않고 1024px 이상 |
| 구도 | 주요 피사체가 잘린 경우 없음 |
| 생성 성공률 | 5회 시도 중 3회 이상 Pass 기준 충족 |

### Good 기준 (목표)

| 항목 | 기준 |
|------|------|
| 조명·색감 정확도 | 프롬프트에 명시한 조명 방향·색감이 표현됨 |
| 분위기 일치 | 방송 제작 맥락에서 바로 쓸 수 있는 퀄리티 |
| 프롬프트 없이도 인식 | 이미지만 봐도 장르·상황이 명확히 전달됨 |

### 평가 방법

- PM 이슬기 or 손찬용이 테스트 프롬프트 세트 5~10개로 생성 시도
- GPT Image 2 vs Gemini Flash 결과 나란히 비교
- 품질 좋은 결과는 라이브러리에 저장해 데모용으로 보관
