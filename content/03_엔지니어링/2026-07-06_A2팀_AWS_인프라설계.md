# A2팀 AWS 인프라 설계
> 작성일: 2026-07-06 | 작성자: 이슬기 (PM)

> **BE 담당자(김유신·송영범) 검토 후 확정 예정**

---

## 서비스 개요

A2팀은 GPT Image 2 기반 이미지 생성과 fal.ai(Kling 3.0 / Seedance 2.0) 기반 영상 생성을 제공하는 AI 미디어 생성 서비스를 구축한다. 생성된 이미지·영상 파일은 S3에 저장하며, 영상 생성은 비동기 방식(request id + polling)으로 처리된다.

## AWS EC2 인스턴스

| 항목 (AWS Console 기준) | 확정 선택 스펙 | 비고 및 기술적 근거 |
|------------------------|-------------|------------------|
| 리전 (Region) | 아시아 태평양(서울) `ap-northeast-2` | 네트워크 지연 시간(Latency) 최소화 |
| 인스턴스 개수 | 1대 (Number of instances: 1) | MVP 단계 단일 서버 구성 |
| 운영체제 이미지 (AMI) | Ubuntu Server 24.04 LTS (ARM64) 또는 Amazon Linux 2023 (ARM64) | Docker 배포 및 운영에 안정적인 OS 환경 |
| 아키텍처 (Architecture) | 64비트(ARM) `arm64` | t4g 가성비 라인업 활용을 위한 필수 선택 |
| 인스턴스 패밀리 | t4g 시리즈 | AWS Graviton 프로세서 기반 가성비 최적화 라인업 |
| 인스턴스 유형 (모델명) | **t4g.small** | 소규모 트래픽 처리를 위한 스펙 / BE 담당자 확정 예정 |
| 하드웨어 제원 | vCPU 2 Cores / RAM 2 GiB | 백엔드 API 서버 및 비동기 polling 처리 구동 가능 |
| 사용자 유형 (Tenancy) | 공유 인스턴스 (Shared) | 비용 최적화를 위한 기본값 멀티테넌트 방식 |

## 스토리지

### 서버 메인 디스크 (EBS)

| 항목 | 확정 선택 스펙 | 비고 및 기술적 근거 |
|------|-------------|------------------|
| 볼륨 유형 (Volume Type) | General Purpose SSD (gp3) | 구형 gp2 대비 비용 10% 저렴, 처리 속도 향상 |
| 스토리지 용량 (Size) | 30 GB | OS 시스템·빌드 파일·로그 적재 공간 |
| IOPS 속도 | 3,000 IOPS (기본 제공 최고치) | 추가 비용 없음 (Provisioned 별도 설정 금지) |
| 처리량 (Throughput) | 125 MB/s (기본 제공 최고치) | 추가 비용 없음 (별도 설정 금지) |

### 미디어 스토리지 (S3)

- **서비스명**: Amazon S3
- **클래스**: S3 Standard (즉시 조회 가용성 확보)
- **초기 산정 용량**: 30 GB
  - 이미지 파일: GPT Image 2 생성 이미지 (1024×1024 기준 약 1~3 MB/장)
  - 영상 파일: fal.ai 생성 영상 (3~15초 기준 약 10~50 MB/클립)
  - 영상 파일 용량이 크므로 B1팀(20 GB) 대비 상향 산정
- **보안 세팅**: 생성된 이미지·영상 URL을 FE에서 직접 렌더링할 수 있도록 퍼블릭 액세스 허용(읽기 전용) 정책 적용
- **고려사항**: S3 특성상 사용량만큼 비용 산정 / 영상 파일 누적에 따라 용량 증가 가능

## IAM 사용자 권한 범위 정의서

### 1. 인프라 생성 및 관리자 권한 (BE 담당자)

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "EC2InstanceManagement",
      "Effect": "Allow",
      "Action": [
        "ec2:RunInstances",
        "ec2:TerminateInstances",
        "ec2:StartInstances",
        "ec2:StopInstances",
        "ec2:DescribeInstances",
        "ec2:DescribeInstanceStatus"
      ],
      "Resource": "*"
    },
    {
      "Sid": "EBSVolumeManagement",
      "Effect": "Allow",
      "Action": [
        "ec2:CreateVolume",
        "ec2:DeleteVolume",
        "ec2:AttachVolume",
        "ec2:DetachVolume",
        "ec2:DescribeVolumes"
      ],
      "Resource": "*"
    },
    {
      "Sid": "NetworkAndSecurityManagement",
      "Effect": "Allow",
      "Action": [
        "ec2:DescribeSecurityGroups",
        "ec2:CreateSecurityGroup",
        "ec2:AuthorizeSecurityGroupIngress",
        "ec2:AuthorizeSecurityGroupEgress",
        "ec2:DescribeKeyPairs",
        "ec2:CreateKeyPair"
      ],
      "Resource": "*"
    },
    {
      "Sid": "S3MediaStorageManagement",
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObject",
        "s3:DeleteObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::a2-media-bucket",
        "arn:aws:s3:::a2-media-bucket/*"
      ]
    }
  ]
}
```

**권한 범위 제한 근거**

- `RunInstances` / `TerminateInstances`: 서버(t4g.small)를 생성하고 삭제할 수 있는 권한
- `CreateVolume` / `AttachVolume`: gp3 30GB 디스크를 만들어서 서버에 장착하는 권한
- `AuthorizeSecurityGroupIngress`: 백엔드 포트(80, 443, 3000 등)를 열기 위한 방화벽 설정 권한
- `s3:PutObject` / `s3:GetObject`: 생성된 이미지·영상 파일을 S3에 업로드하고 URL로 조회하는 권한

### 2. 역할별 IAM 권한 요약

| 역할 | 권한 범위 |
|------|---------|
| BE 김유신 (이미지 담당) | EC2·EBS·S3 전체 / 이미지 생성·저장 API 관련 |
| BE 송영범 (영상 담당) | EC2·EBS·S3 전체 / 영상 생성·polling·저장 API 관련 |
| FE 김은재 | S3 읽기 전용 (이미지·영상 URL 확인용) |
| PM | Billing 조회만 (비용 모니터링) |

## A2팀 특이사항 (B1팀 대비)

| 항목 | B1팀 | A2팀 |
|------|------|------|
| 생성 미디어 유형 | 이미지 | 이미지 + 영상 |
| S3 초기 산정 용량 | 20 GB | 30 GB (영상 파일 용량 고려 상향) |
| 비동기 처리 | 미포함 | fal.ai request id 기반 polling 처리 필요 |
| BE 분담 | 단일 | 이미지 담당 / 영상 담당 2인 분담 |
