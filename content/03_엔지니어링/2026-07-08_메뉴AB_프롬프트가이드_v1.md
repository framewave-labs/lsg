## 1. 개요 — 메뉴 A·B 구분

| 구분 | 메뉴 A (초기 레퍼런스) | 메뉴 B (키이미지) |
|------|----------------------|-------------------|
| 입력 | 간단한 프롬프트 (한 줄) | 디테일 프롬프트 (구조화 형식) |
| 출력 | 배경 이미지 1장 + 캐릭터 다각도(정면·측면·후면) | 캐릭터 디자인 레퍼런스 시트 (턴어라운드 + 표정 + 소품 클로즈업 + 컬러 팔레트) |
| 프롬프트 난이도 | 낮음 — 짧고 단순 | 높음 — PART1(서술) + PART2(JSON 구조) |
| 사용 시점 | 캐릭터·배경을 처음 잡아볼 때 | 초기 결과를 바탕으로 최종 확정 시트를 뽑을 때 |

메뉴 A → 메뉴 B로 이어지는 흐름이며, 메뉴 A 결과(캐릭터 외형 대략 확정)를 참고해 메뉴 B의 디테일 프롬프트를 채워나가는 구조를 권장.

## 2. 메뉴 A — 초기 레퍼런스 프롬프트 가이드

### 2.1 구조 원칙

메뉴 A는 GPT Image 2로 생성하므로, [[2026-07-07_프롬프트가이드_v2]] 2.1의 GPT Image 2 최적화 구조(`[피사체+동작] → [구도/샷타입] → [조명] → [분위기/색감] → [스타일] → [품질 지시어]`)를 그대로 따른다. 한 번의 짧은 프롬프트로 **배경 이미지 1장**과 **캐릭터 다각도 이미지**를 각각 생성하므로, 프롬프트도 배경용/캐릭터용으로 나눠 구성.

**배경 프롬프트 구조 (GPT Image 2 최적화)**
```
[장소/공간]  →  [구도/샷 타입 (establishing shot)]  →  [조명 (시간대·날씨로 표현)]  →  [분위기]  →  [스타일]  →  [품질 지시어]
```

**캐릭터 프롬프트 구조 (GPT Image 2 최적화)**
```
[캐릭터 외형 한 줄 요약]  →  [구도 ("character turnaround, front view / side view / back view")]  →  [조명 (studio lighting, even and diffused)]  →  [단순 배경 (흰색·연한 회색)]  →  [스타일]  →  [품질 지시어]
```

### 2.2 필수 요소

- 캐릭터는 **단순 배경**(plain white / light-gray background) 위에 3뷰(정면·측면·후면) 요청 — 메뉴 B 턴어라운드의 축소판
- 캐릭터 프롬프트는 **조명 키워드를 반드시 포함** (`studio lighting, even and diffused`) — 3뷰 간 조명 불일치로 결과가 어색해지는 것을 방지 (메뉴 B의 LIGHTING 필드 기본값과 동일)
- 배경은 **1장**만 필요하므로 인물·캐릭터 요소는 배경 프롬프트에 넣지 않음
- 필수 끝맺음: `character turnaround sheet, consistent design` (캐릭터) / `establishing shot, no characters` (배경)
- 품질 지시어 — v2 가이드 규칙과 예외:
  - 실사 계열 스타일(semi-realistic, near-future realistic, noir realistic 등)은 v2 규칙대로 **`photorealistic, high detail`** 로 끝맺음
  - 일러스트·애니메이션 계열 스타일(anime, illustration, concept art 등)은 `photorealistic`이 스타일 지시어와 충돌하므로 **`high detail`만** 사용 — v2의 보편 규칙에 대한 명시적 예외
  - 각 프롬프트는 80단어 이내 유지 (v2 규칙과 동일)

### 2.3 피해야 할 것

| 하지 말 것 | 이유 |
|-----------|------|
| 배경 프롬프트에 인물 묘사 포함 | 배경 1장 목적과 불일치, 캐릭터 중복 생성됨 |
| 소품·표정 등 디테일 요청 | 메뉴 A 단계 목적 아님 — 메뉴 B에서 처리 |
| 각도 표현 생략 | "정면/측면/후면"을 명시하지 않으면 단일 각도만 출력됨 |

### 2.5 한국어 자연어 프롬프트 가이드

**배경**
메뉴 A는 사용자가 처음 진입하는 단계로, PD·기획자 등 영어 프롬프트에 익숙하지 않은 사용자가 많을 것으로 예상됨. [[2026-07-07_프롬프트가이드_v2]]에 정의된 **Claude Sonnet 한국어→영어 변환 레이어**를 메뉴 A에도 동일하게 적용하되, 변환 이전 단계에서 사용자가 편하게 입력할 수 있는 **자연어 한국어 프롬프트 예시**를 아래에 별도로 정리함. (번역투가 아니라 실제 사용자가 말하듯 입력하는 문장 형태 기준)

**작성 원칙**
- 완성된 영어 프롬프트 문법을 흉내 내지 말고, **사용자가 실제로 말할 법한 자연스러운 한 문장**으로 작성
- 캐릭터 요청과 배경 요청을 한 번에 섞어 입력해도 되지만, 가이드 문서에는 이해를 돕기 위해 분리 표기
- 각도 표현은 "정면/측면/뒷모습" 또는 "여러 각도에서" 처럼 자연스러운 한국어 표현 사용 (영어처럼 "front view, side view, back view" 나열 강요하지 않음)
- 색상·스타일 지정이 없으면 시스템(Claude 변환 레이어)이 장르 맥락에 맞게 보완하도록 유도하는 문장 허용 (예: "느낌 있게", "분위기 있게" 등 추상 표현도 허용 — 메뉴 A는 초기 탐색 단계이므로 완벽한 디테일 요구하지 않음)

**메뉴 A 한국어 입력 → 처리 흐름**
```
사용자 (한국어 자연어 입력, 캐릭터+배경 구분 없이 자유 입력 가능)
    ↓
Claude Sonnet 변환 레이어 (v2 가이드 1.2 흐름과 동일)
  · 캐릭터 요청 → 캐릭터 턴어라운드 영문 프롬프트로 분리 변환
  · 배경 요청 → 배경 영문 프롬프트로 분리 변환
    ↓
GPT Image 2 (캐릭터/배경 각각 생성)
```

## 3. 메뉴 B — 키이미지 프롬프트 가이드

### 3.1 구조 원칙

메뉴 B는 **PART 1(서술형 설명) + PART 2(JSON 구조화 항목)** 2단 구성을 사용한다. 오전 BE·디자이너 논의에서 공유된 실제 사용 예시([[2026-07-08_오전_논의_전달용]] 참고, "미라 한" 캐릭터 설정표 기준) 형식을 그대로 표준 템플릿으로 채택.

```
### PART 1: PROSE DESCRIPTION
[캐릭터 정체성 + 레이아웃 + 아트 스타일을 2~3문장으로 서술]

### PART 2: JSON STRUCTURE
SUBJECT: [나이·성별·외형·헤어·표정 목록·포즈]
CLOTHING & STYLING: [의상 항목 + 색상 코드 + 소재]
ACCESSORIES: [소품 목록 + 색상 코드 + 소재]
SCENE: [레이아웃 유형 + 핵심 요소]
BACKGROUND: [배경 구성 요소]
LIGHTING: [조명 방식]
TECHNICAL: [카메라·렌즈 정보, 없으면 N/A]
COMPOSITION: [구도 방식 + 초점]
STYLE: [아트 스타일 + 장르 + 대표 색상]
ASPECT RATIO: [비율]
CONSTRAINTS - Must keep: [유지해야 할 핵심 디자인 요소]
CONSTRAINTS - Avoid: [피해야 할 요소]
```

### 3.2 필드 작성 가이드

| 필드 | 작성 요령 |
|------|---------|
| SUBJECT | 나이·성별을 먼저, 헤어 색상은 반드시 HEX 코드 병기, 표정은 감정 단어 나열, 포즈는 "character turnaround showing front, side, and back views" 고정 |
| CLOTHING & STYLING | 상의·하의·신발 순서, 각 항목에 색상 코드 + 소재(fabric) 명시 |
| ACCESSORIES | 상징적 소품 우선 배치, 색상·소재 명시 |
| SCENE | 레퍼런스 시트 레이아웃임을 명시 (`Character design reference sheet layout`) |
| BACKGROUND | 그리드 라인·텍스트 블록·컬러 스와치 등 시트 구성 요소 나열 |
| LIGHTING | 기본값 `studio lighting, even, flat, diffused` — 레퍼런스 시트는 균일 조명이 표준 |
| TECHNICAL | 실사 계열이 아니면 전부 `N/A` 처리 |
| COMPOSITION | `Grid-based reference sheet, Frontal, orthographic, focus on Center character` 기본형 사용 |
| STYLE | 대표 색상 3개까지 HEX 코드로 명시 |
| ASPECT RATIO | 시트형 레이아웃은 `16:9` 권장 |
| CONSTRAINTS | Must keep에 시그니처 요소(엠블럼·헤어·소품) 3~5개, Avoid에 품질 저해 요소(blurry, distorted anatomy 등) 고정 포함 |
