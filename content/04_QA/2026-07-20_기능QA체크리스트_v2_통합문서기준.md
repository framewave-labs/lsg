# A2팀 기능 QA 체크리스트 (통합문서 v0.4 기준)

> 작성일: 2026-07-20 | 작성자: 이슬기 (PM)
> 근거: GitHub `framewave-labs/framewave-docs` — `content/00_통합문서/2026-07-13_A2팀_최종_기능명세서.md` (v0.4, 손찬용 PM 작성) + 첨부 Excel `2026-07-13_A2팀_기능명세서_v0.4.xlsx`(기능명세/추적성/확인필요 시트)

***

**QA 시 원칙:** 통합문서의 완료 기준(정책확인·후속안건·확인 중 항목)을 실제 배포 URL 기준으로 검증하되, 정책 자체가 미확정인 항목은 Fail이 아니라 "정책 미정으로 테스트 보류"로 기록한다.

***

## QA 진행 원칙

* **PM이 시나리오 기반으로 실행**, 버그 발견 시 Severity 분류 후 BE·FE에 전달
* 배포 URL 기준으로 진행 (로컬 환경 아님)
* 데스크톱(1280px) 기준 우선, 반응형은 별도 섹션에서 체크
* 발견된 버그는 디스코드 #a2-QA 채널에 \[버그 번호] \[심각도] \[재현 방법] 형식으로 기록
* **구현 현황(07.13 기준)** 열은 통합문서 Excel의 FE(GitHub 이슈)·BE(노션 진행상황) 기준 — 배포 시점 기준 재확인 필요, ❓ 표기는 통합문서 작성 시점에도 미확인이었던 항목
* 구현 현황이 "노션에 항목 없음"·"구현 보류"·"정책확인"인 항목은 Fail이 아니라 **테스트 보류**로 기록

### 버그 심각도 기준

| 등급       | 기준                       | 처리 기한   |
| -------- | ------------------------ | ------- |
| P0 — 블로커 | 핵심 기능 동작 불가, 배포 전 반드시 수정 | 당일      |
| P1 — 주요  | 기능은 동작하나 UX 크게 저하        | 배포 전    |
| P2 — 보통  | 사소한 UI 오류, 마이크로카피        | 시간 여유 시 |
| P3 — 낮음  | 있으면 좋은 개선 사항             | 향후 고도화  |
